【Event launched when the player touches it】

[Goal]
Let's get to the point where we can use the "Player Touch" event Trigger Condition.

As the name suggests, these events launch when the player character comes into contact with them.
As practice, inside "Sample Map A" in the sample game, let's create
a chicken-shaped event that displays the text "Ko-ke-kok-koo!" when the player touches it.

※ Also use ◆ I want to create a new event as a reference.

[1]
First, use the Map Selector to
・Switch to "Sample Map A".

・Create a new event at a suitable location.

　→ If you are unsure, refer to steps [1]～[3] of ◆ I want to create a new event.
[2]
Let's set the event's graphic. Double-click the area enclosed in a red box.
[3]
A window like the one shown below will appear.
・From the list on the left side of the window, select "Chicken.png".
[4]
The chicken has been set as the event graphic. Next, change the
"Trigger Condition" setting to "Player Touch" so the event launches when the player comes into contact with the character.

This makes the current event launch "when the player comes into contact with the event".
[5]
Finally, create the event contents. Click "■ Show Command Window ■" at the bottom right of the Event window.
[6]
When the "Insert Event Command" window appears, confirm that the top tab shows "Show Message".
・Type '"Ko-ke-kok-koo!"' in the text input field in the center, and click the "input" button.
[7]
The event is now complete. Does the completed event settings look like the following?

1: Graphic is "chicken"
2: Trigger Condition is "Player Touch"
3: The Event Command display on the right shows:
■ Message: "Ko-ke-kok-koo!"
■
[8]
Now save the map and test the event you created. After clicking "Save (Entire Map)", click "Test Play".
[9]
It's hard to tell from a still image, but did
"Ko-ke-kok-koo!" display at the moment the player came into contact with the chicken?

The "Event launched when the player touches it" is now complete. Good work!


The above article was created by copying about 90% from ◆ Event launched when the opponent touches you.
The difference between "Player Touch" and "Event Touch" is: basically, use the former for "events mainly launched when the player touches from their side, such as Location Transfers",
and use the latter for "events mainly launched when the event side touches, such as symbol encounters". That's pretty much the straightforward usage.
Reversing these can make event behavior and player controls feel slightly off — just keep that in the back of your mind.
