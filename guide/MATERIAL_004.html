【Use Face Graphics <※Basic System>】

[Goal]
Face graphics are not just enjoyable to look at —
they allow you to express a character's emotions
more subtly and clearly than character chips,
and they serve to make it clear who is
speaking during a conversation.
They also serve to clearly distinguish
player characters and other important characters
from less important NPCs in the story.
Furthermore, they are easier to create yourself
than character chips or map chips, and easy to introduce.
As long as you can prepare the appropriate asset,
you can display face graphics with a simple configuration.

●About the images used
This time, we will learn how to display the image "FaceGraphic_Wolfarl.png" included in the sample game
as a face graphic in the message window,
and how to display the image "Wolfarl.png" below
as a face graphic on the menu screen and in combat.

The terms of use for this image are at the bottom of this page.
★Note!
　When using assets, always read and comply with the asset's terms of use.
Also see "◆What are the rules for using assets?".
【Steps】

　【Displaying in the Message Window】

1. Select the image file and configure settings
- The settings for this tutorial are done in System Database type 24,
　"Face Graphic Names".
　Open the System Database and select
　type "24: Face Graphic Names".
　There is a data list in the middle of the window.
　Select "4:　" from this data list.

- Next, set the image to use in the window.
　Press the [File] button to the right of the face image filename input field,
　and the folder containing assets will open.
　Select "FaceGraphic_Wolfarl.png" and press
　the "OK" button at the bottom right of the window.
　"Picture/FaceGraphic_Wolfarl.png" has been
　entered as the image filename.

　Wolfarl's image can now be called up.
This is the red-outlined area.
2. Naming the face graphic
- While we're at it, let's give this a name for future management to indicate what kind
　of image file it is.
　The area we will edit is above the data list
　in the middle of the window — the wide input field
　with "ID" written just to its left.
　Selecting data 0–3 from the data list below
　will change the text in the input field.

- Now that you understand roughly where to look,
　let's give a name to face graphic number 4.
　Select "4:　" from the data list,
　and enter "Wolfar" in the input field.
　After entering it, don't forget to press
　"OK" or "Update" at the bottom right of the window to save.
　"4:　" in the data list should now show "4: Wolfar".
　Remembering this will be very helpful.
The blue-outlined area — name things clearly for easy reference.
3. Calling the face graphic in Show Message
- Next, configure settings to call Wolfarl's image.
　Create an event at a suitable position in Sample Map A
　and set the chicken image.
　Open the Command Input window and change the tab to "1 Show Message".

- Before entering, here is one useful piece of information.
　Look at the lower center of the window.
　It says "Face number confirmation (loaded from SysDB 24)"
　with a number input field, and to its right "0: No window display".
　Try setting the number in that field to 4.
　You can type a half-width number directly, use the up/down buttons on the field,
　or scroll with the mouse wheel.
　When the number is 4, "4: Wolfar" should be displayed.
　This is the face graphic name set in steps 2 above,
　and it proves its value right here.
　Conversely, if you didn't name the face graphic,
　or if you gave it a strange name you yourself can't remember,
　you won't know which face graphic it is.

- Now that you've confirmed that Wolfar's face graphic is number 4,
　enter "@4" alone on the first line of the input field and press Enter.
　Use half-width characters. Both @ and 4 must be half-width for the command to work.
　This is the command to "call face graphic number 4".
　You can enter whatever you like on the second line onward.
　But as emphasized, only lines 2 onward are free — the first line is reserved for commands.
　If anything other than @ and a number is on the first line, it won't work as expected.
　The content of lines 2 onward can be anything (even blank), but let's enter:

　　Wolfar
　　"Hello!"

　After entering it, press the "Input" button at
　the bottom right of the window to enter the command.
Pay attention to the bottom!
And the first line is half-width @ and half-width number only!
4. Verify
　Let's do a Test Play.
　When you talk to the event you created, Wolfarl's image should
　appear alongside the message window on the right side.
　Great work!
　Note that the "@(number)" face graphic specification can be set
　each time a message is shown, so alternating numbers will change the
　face graphic accordingly, enhancing the sense of a conversation taking place.
　By default, the chicken image is in "@3",
　so if you have time, try doing the same as steps 2.
It's displaying correctly, right?

The chicken sure looks scared... or maybe not...

　【Displaying on the Menu Screen and in Combat】

1. Save the image file
- Confirm that the image at the top of this article is in the "Picture" folder.
　If it isn't, download it and move it to the "Picture" folder.
2. Select the image file and configure settings
- The settings for this tutorial are done in Changeable Database type 0,
　"Player Character Status".
　Open the Changeable Database and select
　type "0: Player Character Status".
　There is a data list in the middle of the window.
　Select "12: Wolfar" from this data list.

- Next, set the image to use in the menu/combat screen.
　Enter it in the item "Face Image [For Battle/Status]".
　Press the [File] button to the right of the input field,
　and the folder containing assets will open.
　Select "Wolfarl.png" and press
　the "OK" button at the bottom right of the window.
　"Picture/Wolfarl.png" has been
　entered as the image filename. Press
　"OK" or "Update" at the bottom right of the window to save.
　Wolfarl's face graphic can now be called up.
　With this configuration, it will be called automatically,
　which is very convenient.
Set this here and everything else is automatic.
3. Verify
　Let's do a Test Play.
　Open the menu screen or enter combat and the
　face graphic should now be displayed.
　If it doesn't work, review the steps.
　Great work!
In both menu and battle.

[Side Note]

■On the drawbacks of facial expression variations
If you're using face graphics, you'll probably want to create expression variants.
The benefits of expressive face graphics are undeniable, but
creating and managing them is actually several times more work than having just one expression.
Thinking about preparing 5 expressions for each of 4 characters,
compared to the variety of real human expressions, 20 illustrations might not seem like many —
but it's actually quite a lot to prepare.
For a first game, we recommend just one generic expression like a neutral face,
or at most the four basic emotions (happy, angry, sad, happy).
If you do use multiple expressions, naming them clearly is important.
For up to four basic emotions, "chick-happy" alone might work,
but if you have 5–6 expressions per character, naming conventions like "sunset-one-eye-smile-open-mouth" are needed.
Devising systems is itself a skill, and something useful for future projects —
please give it a try through trial and error.
■About face graphic specifications
In truth, there are no firm specifications defined for face graphics.
Depending on the creator's method, many different expressions are possible.
However, without any rules, it can be difficult to know where to start, so
here are some recommended forms as a first step:

- Use "[0] Fit to standard size" in Basic System settings.
- Use dot-style face graphics for supporting characters, created at "24×24".
(Same format as the image displayed in the message window in this tutorial.)
- Use illustration-style face graphics for main characters, created at "96×96 with border".
(Same format as the image used on the menu screen and in combat in this tutorial.)
