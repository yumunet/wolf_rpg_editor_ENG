【I Want to Create Beautiful Maps】

【Introduction】
There are various ways to create beautiful maps,
so it's hard to give a single answer, but here we'll describe
one such method: "how to use layers".

【What is a layer?】
A layer is a feature that lets you draw
the map chips that make up a single map
on separate levels.
There are 4 layers:
　・Layer 1
　・Layer 2
　・Layer 3
　・Event Layer
Of these, Layers 1–3 are mainly used for
placing map chips, while the Event Layer is used for
placing events.
Layers become essential mainly when placing chips
that contain transparent colors.
Layers with smaller numbers are displayed lower (beneath).
Therefore, Layer 3 is displayed on top.

By the way, the image on the right shows
everything displayed on the game screen arranged in order.
Pictures (configurable via events) are at the very top,
followed by events and the player character,
then Layers 3, 2, and 1 below that,
with the "panorama background" displayed below Layer 1.
Below the panorama is solid black.
However, map chips with a passability setting of "▲" or "★"
are drawn above events in some cases.
【How to use layers】
The map on the right is displayed using layers.
Let's look at this map layer by layer.

The 3 images below are Layer 1, 2, and 3 from left to right.
Layer 1 contains stone pavement, grass, and sand,
Layer 2 contains water and the edges of grass,
and Layer 3 contains the goddess statue — each drawn separately.
In the actual game screen, these separately drawn layers
are displayed as a single top-down view image. (top image)
What we'd like you to notice here is that
"chips placed on Layer 2 and above contain transparent areas".
Conversely, chips with no transparent areas
are generally best placed on Layer 1.
【When to use layers】
The key point for using layers is mainly when the chip
you want to place contains a transparent color.
Transparent colors, as the name implies, make the area below visible,
so if there's a chip on the layer below, that chip is shown;
if not, the "panorama background" is displayed.
If no panorama is set, it becomes black.

In the map shown in the image, transparent areas are
created on Layer 1, and if a panorama is set,
it will be displayed there.
The left image has a panorama, and the right image has no panorama.


For example, let's try creating a pond in a grassy area.
We'll use BaseChip.png and WaterChip.png,
which are included in WOLF RPG Editor's full package.
The water chip contains transparent colors around its edges,
so the area below shows through.
Let's make it so that the grass is visible through the transparent parts.

Laying grass chips on Layer 1
and then spreading water chips on Layer 2
will do the trick nicely. (left image)
If you try to do this on Layer 1 alone,
the black areas (transparent color parts) around the water
will be displayed as-is, which doesn't look great. (right image)

Among the chips included in the sample,
all auto-tile chips except for the waterfall
are chips to be placed on Layer 2 or 3.
Even non-auto-tiles such as trees, fences, furniture,
and roofs also contain transparent colors. Place these
chips on Layer 2 or 3 as well.
Note that chips without transparency, such as grass and sand,
are generally better placed on Layer 1.
【I want to see how it looks in the game】
Each layer displays the chips placed on that layer,
while chips on other layers appear faded,
which can make it hard to tell how the actual game screen
will look.
When that happens (though Test Play works too),
try clicking the button labeled "Ev" to
switch to the "Event Layer".
In the Event Layer, you can place events, and also
"view the map from the same perspective as the actual game screen".
Map creation involves frequent checking, so
confirming from the Event Layer is very convenient.
【What if 3 layers aren't enough?】
In WOLF RPG Editor Ver2.00, the maximum
number of layers is 3. However,
it is possible to create a visual effect using
4 layers by combining the Event Layer.

For example, if you want to place
1 [Floor], 2 [Table], 3 [Plate], 4 [Cake],
placing the cake using "Event Placement" on the same tile
can sometimes achieve the desired look.

In the figure on the right, a cake is placed on the Event Layer
to represent "placing a cake on a plate on a table in an inn".
When using this method, if you find that what you placed via event
appears behind another chip in the actual game screen,
please check the passability settings of the map chip.
If any chip is set to ▲ or ★,
those chips are drawn above events.
Here are solutions for when you want the event-placed item
to be displayed in front of everything.

[1] When there is a ▲ chip
Enable the "Above Player Character" setting in the event's settings.
This will cause the event-placed chip to be displayed at the very front.

[2] When there is a ★ chip
This case is trickier.
There is currently no way to draw an event above a ★ chip.
If it doesn't cause any issues, you can work around it by changing
the problematic chip's passability setting to ▲ or similar.


For details on passability settings ▲ and ★, please see
Change Map Chip Passability Settings in the Perfect Guide.
【About Auto-Dungeon Map Generation】
WOLF RPG Editor includes an automatic map generation system provided by Het.
Since it specializes in creating intricate and complex maps,
it is a very convenient tool especially for creating dungeons.
The map shown in the image on the right was generated using this system.

Now let's actually try using auto map generation.
[1] First, create a new map. Near the top of the screen,
there is an "Auto Map Generation" icon, so click it.
[2] Clicking it opens a window like the one on the right.
Auto map generation consists of 4 chip types:
"Ceiling", "Wall Upper", "Wall Lower", and "Floor".
Regarding "Wall Upper" and "Wall Lower":
in auto map generation, all walls are 2 chips tall,
made up of "Wall Upper" for the top half and "Wall Lower" for the bottom half.
The tile 1 row above a wall is always "Ceiling", and the tile 1 row below is "Floor",
but in other locations, "Floor" is drawn in straight lines of random lengths.
All areas that are neither wall nor floor are filled with "Ceiling".
[3] Now let's register the chips to use.
Try registering the same chips as shown in the image for
"Wall Upper", "Wall Lower", and "Floor".
The chip designated for "Floor" is located in the
position 3 rows from the top and 2nd from the left in the chip set.
Do not register anything for "Ceiling".
If you accidentally registered something, checking "No Drawing" will clear it.
[4] Once the chip settings are done, let's try generating right away.
First, just click "Generate Map" as-is.
The map won't be drawn to the map yet. The generated result
will be displayed in the "Preview" pane on the left.
(The figure in the image on the right is just an example.)
[5] Each time you press "Generate Map", a different layout is generated,
so feel free to try it several times.
Clicking "Fill With This" at the bottom-right of the window
will fill the map according to the generated layout.
[Side Note]
We haven't mentioned it yet, but let's briefly talk about
the "Target Layer", "Wall Density", "Complexity",
and "Smooth Shapes" options between the chip settings area
and "Generate Map".

・"Target Layer"
As it says, this specifies which layer to generate the map on.
Layer 1 is recommended in general, but as mentioned
earlier in this article, if you want to create a map using
chips that contain transparent colors, use Layer 2 or 3.

・"Wall Density"
"Wall Density" is perhaps better described as "Ceiling Density".
Setting it low increases the amount of floor on the map.
Try creating a slightly larger map and then comparing 10% and 90%
— the difference should be very clear.

・"Complexity"
Setting this complexity low reduces walls between floor areas,
making the map simpler. Setting it to "Most Complex" creates a
highly intricate maze-like layout. However, it is heavily influenced
by "Wall Density", so if it's not as complex as expected,
try reducing the wall density.

・"Smooth Shapes"
Checking this box and generating the map will cause
some walls and floors to be drawn with diagonal lines.
Without the checkbox, the map is drawn entirely with
straight lines that intersect at right angles (all corridor corners are 90 degrees).
