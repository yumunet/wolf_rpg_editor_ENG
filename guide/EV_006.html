【A treasure chest that stays open once opened】

[Goal]
　Inside "Sample Map A" in the sample game, create a treasure chest that stays open once opened.
　The specific goals are listed below.

　　・ Examining it lets you obtain an item
　　・ After obtaining it, the graphic changes to "open treasure chest"
　　・ Examining it afterwards displays "It's empty." (prevents obtaining the item repeatedly)
→
　Like this.

　It is a bit complex, but it is an essential technique, so let's learn it.

[1]
　First, prepare the event.

　・Use the Map Selector to switch to "Sample Map A".

　1. Create a new event at a suitable location.
　
　2. Set the graphic to "closed treasure chest".
　

　→ If you are unsure,
　　　refer to steps [1]～[4] of ◆ Event launched by Confirm key.
[2]
　Next, create the event contents.
　
　Click "■ Show Command Window ■"
　at the bottom right of the Event window.
[3]
　The "Insert Event Command" window should appear.
Let's create the process for obtaining an item.
　
　1. Find and click "Other 2" from the top tabs.

　2. Confirm that the circled "Event Insertion" is selected.
　　　If the white circle to the left of the text has a black dot in it, it is selected.
　　　　※ If it is not selected, click it to select it.

　3. Clicking the square-enclosed area will display a list of
　　　Common Events you can choose from.
　　　Since this is an item-obtaining event,
　　　select "Common 0: ○ Item Increase/Decrease".
　
[4]
　Next, configure the Common Event details.
　Edit the "Common Ev Input (value)" area.
　Follow the descriptions to enter values.
　The description in this area varies by Common Event.
　
　1. In "Item Number",
　　　select the item to obtain.
　　　This time, select "Medicine Bottle".
　　　　※　The notation "[User 2]" at the end
　　　　　　　can be ignored.
　
　2. "Increase/Decrease Amount" sets
　　　how many of item 1 to increase or decrease.
　　　Setting a negative value will decrease
　　　the item count,
　　　but here we naturally want to increase it.
　　　Set it to [1].
　
　3. "Obtain Message" is the field that sets
　　　whether to display a message saying
　　　"Obtained X [item name].".
　　　You could skip it here and display it later with "Show Message",
　　　but let's use it since it's available.
　　　Select the counting unit from the list.
　　　Let's go with the standard "X item(s)".
　　　Select "1: Yes [unit: ~items]".
　
　4. Finally, press the arrow "input" button.
　

　Note: keep the "Insert Event Command" window open
　as we will use it again.
　
　　　So, after all that, the
　　　process to obtain 1 of the item "Medicine Bottle"
　　　is now complete.
[5]
　Now handle the variable.
　Don't worry about the purpose of this process for now;
　just configure it as described below.
　※ It will be explained later, so don't worry.
　
　・First, from the top tabs of the window
　　you kept open,
　　find and click "Variable Operation".
　　
　・You should now see the same screen as the one on the right.

　・We'll skip the detailed screen description, sorry.

　　have no checkmarks.
　
　・Check the areas highlighted in red.
　　Configure them to match the image on the right.

　By the way, what was configured here is:

　"Assign the answer of

　Close the "Insert Event Command" window.
　If it looks like the figure below, you succeeded.
　
　
[7]
　So far, the "event to obtain an item" is complete.
　However, in the current state, you can obtain the item any number of times.
　
　To prevent obtaining it more than once, switch the event's page.
　
　Page 1 will have the "closed treasure chest" to obtain the item,
　and page 2 will have the "open treasure chest" that displays "It's empty.".
　
　When the item is obtained from the chest, we record that "the chest was opened".
　That record is the "Self Variable 0" = [1]+[0] we set earlier.
　
　By setting page 2 to appear when "Self Variable 0" is [1],
　the event will run page 1's content until the chest is examined,
　and page 2's content after the item has been obtained.
　
　The description became a bit difficult — apologies,
　but it is fine if you don't fully understand right now. It will make sense with experience.
　
　Anyway, let's move on.
[8]
　Look at the upper left of the Event window.
　Pay attention to the area enclosed in the yellow box.
　
　These are buttons to add, delete, and
　switch between event pages.
　
　Each page has something called
　"Event Trigger Condition",
　shown in the area enclosed in the blue box below.

　This "Event Trigger Condition" means
　"existence condition" or "appearance condition".
　When the condition is met, that page
　exists (appears).
　
　
　Now, create page 2.
　
　1. Click the "New Page" button.
　
　2. Click the "Page 2" button to
　　　open page 2.
[9]
　A new page with no graphic or event content
　has been created.
　
　Page 2 is the state of "after the chest was opened".
　Set the graphic to "open treasure chest".
　
　1. Double-click the square.
　
　2. Select (Tile Set).
　
　3. Select "open treasure chest".
　
　Then press the "OK" button.
[10]
　Set the "Event Trigger Condition".
　Pay attention to the area enclosed in the purple box.
　
　1. Click the leftmost checkbox to
　　　check it.
　
　2. Click the "Self" button to select it.
　
　3. Select "Self Variable 0".
　
　4. Enter the value [1].
　　　You can also click the ▲ on the right to set it.
　
　5. Select "Is equal to".

[11]
　Set the event content.
　
　・Click "■ Show Command Window ■".
　
　・Select "Show Message" from the top tabs.
　
　・In the input field enclosed in the box,
　　enter "The treasure chest is empty."
　
　・Click the "input" button.
　
　
　The event content for page 2 is just this.
[12]
　Let's summarize.
Page 1:
　Graphic
　　"Closed treasure chest"
　Event content
　　・Obtain an item
　　・Set "Self Variable 0" to [1]
　Event Trigger Condition
　　（None specified）
 　Page 2:
　Graphic
　　"Open treasure chest"
　Event content
　　・Display "It's empty."

　Event Trigger Condition
　　"Self Variable 0" equals [1]
　

Let me explain how this event works...

　At the start of the game, all Self Variables are [0].
　Page 2's Trigger Condition is not met, so it does not appear.
　Therefore, page 1 — which has no condition specified — appears as this event.
　
　Running page 1's event obtains the item and
　sets "Self Variable 0" to [1].

　Then, page 2's Trigger Condition is met,
　so from that point on this event will run page 2.
　
　
※ Note that pages with higher numbers take priority.
　　If multiple pages have their conditions met,
　　the last page appears.
　
　　In this event too, both page 1 and page 2 have their conditions met.
　　(No condition = condition is always met.)
　
　　If that feels uncomfortable, it is fine to add a condition to page 1 as well.
　　Example: "Self Variable 0" equals [0]

【Supplementary Lesson】 What are Self Variables?

 　　Self Variables are variables that an event possesses.
　　They are basically used to record the state of that event.
　　(Such as "is it open?" or "talked X times")
　　
　　Like the treasure chest in this example, you can use them as a flag for "whether it was opened",
　　or you can record how many times the player has talked to that event.

　　Every event has 10 of them each.
　　You can reference them from other events or modify their values,
　　but it gets complex so it is not generally recommended.
　　
　　It is recommended to use "Normal Variables" for variables shared between events,
　　and "Self Variables" for variables used only by that event.
　　
　　These become very convenient variables once you are used to them.
　　Try various things and learn them well.
