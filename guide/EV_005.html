【Event that never reappears once it has disappeared】

[Goal]
Next, create an event that disappears after triggering and stays disappeared even if you leave the map and return.
As an example, create a "chicken that disappears when spoken to and never comes back".
An event that disappears when talked to and remains gone even after you move to another map and come back.
※ Also use ◆ I want to create a new event as a reference.
[1]
・Using ◆ Event that disappears after the event and comes back when you return to the map as a reference,
create "a chicken that gradually disappears when spoken to".
　　
[2]
With this setup, the chicken will revive when you switch maps. To make it permanently disappear,
perform "Self Variable Operation" and "Trigger Condition settings".
[3]
First, perform the "Self Variable Operation".
Display the "Command Input Window" and select "3. Variable Operation" from the left list.
For now, don't worry about the meaning of each item — just click in order: 1, then 2.

↓ Enlarged view of part
[4]
Was it entered as follows? Be careful that ■ Variable Operation: This Ev's Self Variable 0 = 0 + 1 comes first.
If it is in the wrong order, right-clicking a command will allow cut and paste operations.
[5]
Next, click the checkbox in the "Trigger Condition" area.


By checking this box, this event becomes one that "only launches when the value entered in 'Self Variable 0' is '0'".
Though we say "launch", it is easier to understand if you think of it as the "event's existence condition".
If you are confused, just think of it as "set a switch — when it's on the chicken exists, when it turns off the chicken disappears".
※ The "0" at the end of "Self Variable 0" represents the variable type, not the value entered in the variable.
[6]
Let's test it. Click "Save (Map)", then "Test Play" in that order.
[7]
Did the chicken disappear when spoken to, and stay disappeared even after switching maps?
By talking (pressing the Confirm key), the event ran and "Self Variable 0" changed to a value other than "0",
so the event existence condition switch turned off and the chicken disappeared.
If "it doesn't disappear" or "it revives when switching maps",
review the command order, the value of "Self Variable 0" (Trigger Condition = 0, command = 1), etc.
This completes it. Good work!


"The chicken disappeared, but it disappeared instantly rather than gradually."
If you thought that, you are sharp. Actually, in the event above, when "Self Variable 0" was modified, the chicken was completely erased,
so the "Temporarily Erase Event" process was never executed.
But if you put "Temporarily Erase Event" first, then the Variable Operation won't execute, and the chicken will revive every time you switch maps.
There is of course a way to solve this and "slowly, permanently erase the chicken",
but it becomes somewhat complex, so it is summarized in ※ Reference: Slowly and permanently erasing + Common Event basics.
What is done at the link above is complex, so only try it if you really want to make the chicken disappear slowly. The explanations are also somewhat sparse.
If you don't mind the chicken being invisible from the start, or if you looked at the link but couldn't figure it out, feel free to skip it.


The "Self Variable Operation" in events is used very frequently when implementing more complex processes.
