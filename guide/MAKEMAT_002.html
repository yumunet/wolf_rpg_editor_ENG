【How Should I Arrange Walking Character Assets?】

【Question】
When creating walking graphics for use in WOLF RPG Editor yourself, do you ever wonder how they should be arranged?
Here, we answer that question.

Since WOLF RPG Editor can handle various types of walking graphics, it can sometimes be confusing.
Let's briefly touch on the specifications here.

【Solution】

[0] First, about size
　There are actually no restrictions on size.
　For example, for an 8-direction, 3-pattern asset, as long as the width is a multiple of 6 and the height is a multiple of 4, any size is fine.
　In other words, you can even create a character chip that greatly exceeds the width or height of the original map chip!
　However, this makes passability settings and similar configurations more complicated, so it's generally best not to go too large.
　Until you're more comfortable, try creating assets using the character chip sizes included in the basic set as a reference.
[1] Now to the main topic: character placement for 8-direction, 3-pattern assets
　The character placement goes, from top-left downward: "Down", "Left", "Right", "Up",
　and from top-right downward: "Lower-Left", "Lower-Right", "Upper-Left", "Upper-Right" facing directions.
　Please refer to the figure on the right.

　The facing directions indicated by the arrows in the figure are the facing directions displayed during gameplay.
　(Example: For the Left-facing direction image, the image in the second row from the top in the left column, with the left arrow, is used)

　Regarding image animation, the 3 images arranged horizontally are used.
　During animation, images are displayed in the order B->A->B->C->B->...
This is the layout. Compare it with the assets you have on hand.
[2] About 4-direction assets

　For 4-direction assets, the facing directions are "Down", "Left", "Right", "Up" from top to bottom, as shown here.
　Using only the left half of an 8-direction asset gives you a 4-direction asset. Try comparing them.

　If you want to switch between using 4-direction and 8-direction assets, please refer to "◆ I Want to Switch Character Assets from 4-Direction to 8-Direction".
Compare it with the specification above.
＊Note!
　Within a single game, 4-direction assets and 8-direction assets cannot be used simultaneously.

【Details】

[3] Special specification: T
　Files with a filename ending in "T.png" have a different specification.
　For each facing direction, one additional image is added to the left.
　In-game, when the event using this asset is standing still, the added image is displayed.
　In the description figure, during movement it displays B->A->B->C->B->..., and while stopped it displays D continuously.
One still frame added for each facing direction.
[4] Special specification: TX
　Files with a filename ending in "TX.png" have an even different specification.
　For each facing direction, the same number of additional images as used in the animation are added to the left.
　In-game, when the event using this asset is standing still, the added images are used for animation.
　In the description figure, during movement it is B->A->B->C->B->..., and while stopped it is b->a->b->c->b->...
Additional animation frames while stopped added for each facing direction.
[5] Relationship with RPG Maker specifications
　Actually, the layout for 4-direction walking graphics is the same as in RPG Maker XP and later versions.
　Assets based on those specifications can be used directly in WOLF RPG Editor. Of course, always respect the terms of use!

　Also, with WOLF RPG Editor's "Variable Operation+" command, you can retrieve the facing direction of an event,

　Assets based on RPG Maker 200x cannot be used as-is because the facing directions differ.
　(Example: Walking graphic assets distributed by First Seed Material)
　The facing direction order is "Up", "Right", "Down", "Left".
　Either edit the image yourself to match the facing directions above, or use a conversion program such as "Tkool2Wolf".
　However, when modifying, please check the asset's terms of use. (If modification is prohibited, you cannot do so.)
