【◆ An error appeared saying the map chip must be 128 or 256 pixels wide】

【Situation】

This introduces the error about map chips needing to be 128, 256 (or 320) pixels wide.
You will encounter this error when registering a map chip. (See "◆ I want to use a different map chip asset in my game")
The items to check are described below.
This is the kind of error you'll see.
[1] What is the size of the asset you are trying to use?
When registering a map chip, check the horizontal size of the base map.
Type A: Horizontal size is 128 pixels
Type B: Horizontal size is 256 pixels
Type C: Horizontal size is 320 pixels
(Type X: None of the above)
[2] What is the current resolution setting?
What is the resolution setting (from Game Basic Settings)?
Type a: Set to 320×240
Type b: Set to 640×480
Type c: Set to 800×600
[3] What combination of the two types above did you get?
A with a, B with b, and C with c are the correct combinations, so no error will occur.
Conversely, any other combination will trigger this error.

【Solution】
Change the Game Basic Settings to avoid the error.
The simplest solution is to change the resolution setting in Game Basic Settings to create a "correct combination".
This will avoid the error.

[Side Note]

◆ If Type X applies to you...
One of the following may be the case:
A: You are trying to register a RPG Maker-format asset as-is.
B: You created a custom map chip with the wrong size.
C: You are trying to register an auto-tile as a map chip.

In case A, use the asset conversion tool "tkool2WOLF+" to convert it into a format usable in WOLF RPG Editor.
For details, see "◆ I want to use RPG Maker-sized assets in WOLF RPG Editor".
Also please read "◆ What are the rules for using assets?" before using it.

In case B, verify the horizontal size and recreate the asset.
For reference, see "◆ How should I create map chip assets?".

In case C, confirm where you should actually be registering it.
Among the images included in the sample, files with [A] in the name are auto-tiles.
Images that should be registered as map chips have [Base] in the name.
◆ Similar error for auto-tiles
For auto-tiles, no error will appear.
However, the display may not work as intended.
Check whether the standard matches by checking the vertical size as follows:
Type A: Vertical size is 80 pixels
Type B: Vertical size is 160 pixels
Type C: Vertical size is 200 pixels
(Type X: None of the above)
The resolution setting is the same as in [2] above in 【Situation】.
