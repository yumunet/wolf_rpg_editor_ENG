【"Processing exceeded 500,000 operations per frame!" appeared】

[Goal]
Resolve the error "Processing exceeded 500,000 operations per frame!"

Learn the cause of and solutions to this error message, which you will encounter fairly often once you get a bit more familiar with WOLF RPG Editor.
Once you understand it, you will be able to respond quickly next time.

In WOLF RPG Editor (as of Ver.1.31), the limit for processing within one frame (= 1/60 of a second) is 500,000 operations.
In other words, this error occurs because more than 500,000 operations are being performed within a single frame.
So why does this error happen?
【When stuck in an infinite loop】
An "infinite loop" is when, inside a loop,
you cannot exit it and the same processing repeats indefinitely.
As a result, the number of operations within a single frame often exceeds 500,000.
The main causes are the following two:
① There is no "Break Loop" in the Loop
Because the loop is never interrupted,
it loops infinitely, far exceeding 500,000 operations.
Adding a "Break Loop" at the appropriate position in the loop
may resolve the error.
◆ To input "Break Loop":
・Open the map event editor,
　click "■ Show Command Input Window ■".
・Switch the top tab to "Other 1".
・Click "Break Loop" slightly below the top tabs.
【One Point!】
Also consider the case where the loop condition cannot be met,
so "Break Loop" never executes.
Check whether the conditions (greater than or equal to, less than, greater than, etc.) are correct,
and whether the variable used in the Conditional Branch is correct.
② Auto-execute Event
Auto-execute events have the property of
"automatically repeating from the beginning even after they finish running",
so they can also become a kind of infinite loop.
In this case, since the cause is an "Auto-execute" event rather than a loop,
use "Temporarily Erase Event" instead of Break Loop.
◆ To input "Temporarily Erase Event":
・Open the map event editor,
　click "■ Show Command Input Window ■".
・Switch the top tab to "Other 1".
・Select "Temporarily Erase Event", the third from the bottom in the lower section.
・Set the target to "This Event" and enter [0] for the fade.
・Click the "input" button when done.
【One Point!】
In this case too, you can think of it as:
the auto event's start condition continues to be met,
so the event keeps launching.
Therefore, in addition to erasing the event itself,
you can also exit the loop by making the condition no longer met
at the end of the event.
【When processing simply overflows】
This often occurs when using parallel events in action games, etc.
The main countermeasures are to insert waits at appropriate places, or to reduce the number of operations.
◆ To input "Wait":
・Open the map event editor,
　click "■ Show Command Input Window ■".
・Next, switch the top tab to "Other 1".
・Select the "Wait　　Frame" option, the second from the bottom in the lower section.
・Finally, click "input" in the lower right
　to insert a 1-frame wait into the event.
