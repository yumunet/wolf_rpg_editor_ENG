[What exactly is the Basic System?]

[Goal]
　What is this "Basic System" built into WOLF RPG Editor?
　What role does it play in your game? That is what we learn here.

[1]　What can you do with the Basic System?

　First, here is a summary of what the
　Basic System provides for game creators:

　· Set player character names and stats
　· Form a party (up to 6 members)
　· Configure skills (magic/spells)
　· Configure items
　· Includes a money system
　· Configure enemy characters
　· Open the menu with the Cancel key
　· Trigger front-view battles
　· Trigger shop events
　· Automatically display a message window image when showing messages
　· Display Save/Load screens

　However, the following are not part of the Basic System
　and are instead built as "events":

　· Displaying the title screen
　· Choosing "Start" / "Continue" on the title screen
[2]　Which data is the Basic System?

　In the sample game, the Basic System Ver3 data consists of:

　· User Database Types 0–17
　· Changeable Database Types 0–19
　· Common Events Commons 0–209
　　(※It is also recommended to keep Common 213 "Game Over Event" initially.)

　These three sections make up the "Basic System".

　All other "maps" and "events" are not part of the Basic System,
　so deleting them will not affect it.
[3]　What happens if you remove the Basic System?

　If you try to make a game with "completely blank data"
　that has no Basic System, you will have:

　· No message window image when showing text
　· No menu when pressing the Cancel key
　· No Common Event to trigger battles
　· No Common Event to trigger shops
　· No Common Event to add money or items
　· No place to enter player character names and stats

　In other words, the game would only let you
　"walk around a map and activate events with the Confirm key".
　Before the Basic System existed,
　WOLF RPG Editor projects had to start from this state.

　Eventually, building your own custom system is a worthy goal, but until you
　no longer need a guide like this one, we strongly recommend
　making use of the Basic System.
