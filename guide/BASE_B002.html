【[Battle] I want to add more enemy characters】

[Goal]
We will create a new enemy character, register it in an enemy group, and then fight it in combat.

This time, we will use the enemy character graphic "GD_Bahamut(Silver)" included in the WOLF RPG Editor full set
to put a new enemy character "Dragon" into combat.

[1] Clicking the User Database will
display a window like the one on the right.
[2] When the window first opens, the "Type" is set to "skill".
Please click "9:Enemy Character Individual Data".
[3] Click on an empty slot in the "data" list.
[4] Enter values for the "Page 1" entries.
First, enter the following 3 entries that are the minimum required for operation.
• Enemy character name
Enter "Dragon".
• Max HP
Enter "2500".
Any integer of 1 or more is fine.
• Image file
The image file is where you register the appearance of the enemy during combat.
Clicking "File" to the right of the image file field
will bring up a "Select File" window like this.
Clicking "[Picture]" in the top left will show a list of folders
inside the Data folder.
Click "[EnemyGraphic]", which contains enemy graphics.
Many pictures will be displayed;
click "GD_Bahamut(Silver)" at the very top,
then click "OK" at the bottom of the window.

The image filename will then be registered in the
"image file" field of the User Database editor.
[5] After confirming that the enemy character name, max HP, and image file have all been entered,
click "OK" at the bottom of the window to close it temporarily.
This saves the work you have done so far in WOLF RPG Editor.
Next, prepare to put the enemy into combat. Referring to ◆ I want to configure an enemy group,
enter the "Dragon" you just created (1 unit) as Enemy 1 (Center).
[6] Continue following ◆ I want to configure an enemy group
to create an event that triggers a combat.
You're almost done — just a little more to go!
Set "Enemy Group to Appear" to "9:Dragon",
and once the event is done, do a Test Play.
If a large white dragon appears, you have succeeded!

If it is not a Dragon,
the "Enemy Group to Appear" setting in the event may be wrong,
or the enemy character registered in the enemy group may be incorrect.
Please check. Good work!
