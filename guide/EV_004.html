【Event that disappears after the event and comes back when you return to the map】

[Goal]
　Create an event that disappears after triggering, but comes back when you leave the map and return.

　As an example, create a "chicken that disappears when spoken to, but comes back when you return".
　Nothing happens when you talk to it — it just disappears,
　but when you move to another map and come back, it has revived.

[1]
　・First, use the Map Selector to switch to "Sample Map A".

　・Create an event at a suitable location
 　and set the chicken image.
　　（Place it at the position shown in the image on the right）


 → If you are unsure, refer to
　  ◆ I want to create a new event's
　　steps [1]～[5].
[2]　
　Click "■ Show Command Window ■"
　at the bottom right of the Event window.
[3]　
　・When the "Insert Event Command" window appears,
　　　select "Other 1" from the top tabs.

　1. Select "Temporarily Erase Event" near the bottom.

　2. Select the event to temporarily erase.
　　　Confirm that the target is [ This Event ].
　　　※ By specifying here, you can also temporarily erase
　　　an event other than the currently running one.

　3. Specify the fade duration.
　　　Enter [60] in the input field next to "Fade".

　　　Note that a fade time of [0] is used most of the time.
　　　Here we specified a value to make the effect noticeable.

　　※　The "fade" here means
　　　　"gradually disappearing".
　　　　With Fade [0], it disappears instantly,
　　　　and with [0] or higher, it gradually disappears.

　When done, press the "OK" button.


　What we specified here is the command:
　"Temporarily erase this event,
　　fading out over 60 frames."

　This is the only command to enter this time.
[4]

Does it look like the figure on the right?

　Now save the map and
　test the event you created.

　After clicking "Save (Entire Map)",
　click "Test Play".
[5]

Talk to the chicken you created.
Nothing happens.
However, the chicken's graphic starts becoming transparent.
Image 1 shows it mid-way — it is semi-transparent.

After 60 frames (approximately 1 second), the chicken has completely disappeared.
You can now walk through the spot where the chicken was,
and clicking at that spot produces no response.
The event has disappeared. (Image 2)

Now, move to a different map and come back.
Going all the way up will take you to a different map,
so go there once and come back. (Images 3 and 4)

You will see that the chicken that disappeared earlier has revived. (Image 5)
Talking to it again will make it disappear over 60 frames just as before.
And every time you move away and come back, it will revive.


This completes it.
Good work!


Note that "Temporarily Erase Event" is only temporary, as you saw —
if you move to a different map and return, it will revive.

The method to completely remove it (make it never launch again) is
described in a separate article "◆ Event that never reappears once it has disappeared",
so please see that one.

[Side Note]

In this article we used it with a Confirm key-launched event to make the effect noticeable,
but "Temporarily Erase Event" is frequently used in auto-execute events.

For details, see the auto-execute event page.
