【I want to detect whether multiple keys are pressed simultaneously】


　By changing the assignment destination of key input, you can simultaneously check whether multiple keys have been pressed. 　

[Goal]

　1. Use key input to check whether multiple keys are pressed simultaneously.
　　(This time we will create an event that detects simultaneous pressing of Left Shift and the Confirm key.)
　　
　2. Display a message when the keys are pressed simultaneously.

　
　【Related topics】
　　I want to detect whether a specific key has been pressed
　

【Steps】

―― [1] Create the event ――

　First, switch to "Sample Map A" using the Map Selector.
　Create an event that launches with the Confirm key in an easy-to-find location.
　Set the graphic to a chicken.
　(Reference: 【Event that launches with the Confirm key】)
　

―― [2] Loop command ――　

　Click the
　"■ Show Command Input Window ■" button at the bottom-right of the Event window
　to open the Event Command Input Window.
　
　Select "H Event Control" from the list on the left.
　Click "Loop (Repeat)" near the upper-left and enter it.
　
　
　※ The reason we use a Loop this time is because this is an event launched with the Confirm key.
　　　Also, since we are using two key inputs, we cannot check "Wait until key is pressed"
　　　(as the process would get stuck on one of them).
　　　Therefore, for parallel events that run every frame, a Loop is not necessary.
　

―― [3] Key input 1 (Left Shift) ――

　Enter commands inside the Loop.
　Select "9 Key Input" from the left side of the window.
　
　Leave the assignment destination as "This Ev: Self0" without changing it.
　Click the key type and select "All Keyboard Keys (100+)".
　Check "Detect Specific Key Only" just below it,
　and enter "142" in the key code field.
　
　Also confirm that "Wait until key is pressed" is not checked.
　
　Finally, click the "OK" button at the bottom-right of the window.
　
　※ The key code values are listed under
　　　Event Creation → Key Input in the manual.
　　　You can also enter them in the editor and the corresponding key name will be displayed at the bottom.
 　

―― [4] Key input 2 (Confirm key) ――

　Now enter the second key input command.
　
　In the same key input setting, change the assignment destination.
　Select "This Ev: Self1" as the assignment destination.
　
　Then change the key type back to "Basic".
　Check only the Confirm key entry and uncheck everything else.
　The direction key is also not needed, so set it to "Do Not Accept".
　
　When done, click "OK" at the bottom-right of the window.
　

―― [5] Configuring the Conditional Branch ――

　This time select "7 Condition (Variable)" from the list on the left.

　Uncheck "Link 3 Variables".
　Set the 1st Conditional Branch to "This Event: Self0 equals 0"
　and the 2nd Conditional Branch to "This Event: Self1 equals 10".
　
　Leave the inside of the 1st branch empty and don't touch it.
　Inside the 2nd branch, use "Show Message" to enter the message
　to display when the keys are pressed simultaneously.
　
　※ Explanation of this branch is provided at the bottom of the page

―― [6] Adding a wait ――

　Finally, move the cursor to between
　"End Branch" and "End Loop".
　
　Click "H Event Control" from the left side of the window again.
　Click the Wait item and enter 1 frame.
　

　This is now complete.
　Save and do a Test Play.
　
　
　When you talk to the event your movement is disabled,
　but that is not a problem.
　
　First, press only the Confirm key.
　Confirm that the message is not displayed with Confirm key alone.
　
　Next, press the Confirm key while simultaneously holding Left Shift.
　If the message you entered is displayed, you succeeded!
　Great work.
　
【If a problem occurs】

◆ Talking to the event shows a green banner saying "Processing exceeded 500,000 per frame!"
　You forgot to add a wait, or the wait is in the wrong position.
　Please verify.
　
◆ Pressing only Left Shift or Confirm key causes the message to be displayed
　The Conditional Branch settings are wrong,
　or the message may be inserted in the wrong position.
　Review the branch settings and the position of the message.
　
◆ When a key is pressed, it only responds occasionally
　The wait value may be too large.
　To check every frame, set it to 1 frame (0 for parallel) or close to that value.
　
◆ Simultaneous key press produces no response
　　· The key input assignment destination is wrong
　　· The Conditional Branch settings are wrong
　　· The key code value is wrong
　are possible causes.
　Verify each one.
　
[Side Note]

What is the Conditional Branch in [5] doing?

　When the correct key is pressed, its value is assigned to the assignment destination variable.
　In this case, Self0 gets Left Shift's value "142",
　and Self1 gets the Confirm key's value "10".
　At all other times, "0" is always assigned.
　
　In other words, the 1st branch is satisfied when Left Shift is not pressed.
　Therefore, when Left Shift is pressed the condition is not met, and the check proceeds to the 2nd branch and beyond.
　
　By writing the Confirm key condition in the 2nd branch,
　you can detect the state where Left Shift is pressed and the Confirm key is also pressed
　using a single Conditional Branch command.
　
　Of course, clarity is the first priority when creating,
　so it is fine to use two separate branches as shown in the image.
