【I want to set an enemy's resistance to status ailments】

　[Goal]

　Status ailments affect characters during combat, causing HP to decrease each turn,
　putting them to sleep so they cannot act, or preventing them from using magic.

　It is fun to create strategic battles by sealing a magic-type enemy's magic
　or slowing down a fast beast-type monster,
　but it would be a bit embarrassing if a powerful boss standing in the player character's way
　got paralyzed and became unable to move, then got beaten up.

　In this topic, we will learn how to configure status ailment resistance by
　strongly setting the enemy character "Goblin" (already included in the sample game)
　to resist the Paralysis status.

　【Steps】
[1] Opening the User Database

　Clicking the User Database will
　display a window like the one on the right.
[2] Selecting the type to edit

　When the window first opens, the "Type" is set to "skill".
　This time the settings will be made in User Database entry 12 "┗ Status Resistance", so
　please click "12:┗ Status Resistance".
[3] Selecting the data to edit

　There is a data list in the middle of the window.
　Please select "31:Goblin" from this list.
[4] Resistance against elements

　Next, configure the probability of status ailments taking effect.
　This time we want to strongly resist the "Paralysis" status, so
　we will edit "<Paralysis> Success Rate" on the right side of the window.
　This is the 5th entry from the top on Page 1.

　Enter "0" (completely ineffective) in the input field.
　Once done, click the "OK" or "Update" button at the bottom right of the window to save.
[5] Test Play

　Once you have configured the status ailment resistance, let's do a Test Play.
　If you want to do a Test Play but aren't sure how to create a combat event,
　refer to "I want to trigger a battle"
　to call enemy group entry 31 "Goblin x1" from a map event or similar.

　If you try using the default skill "Paralyze" on the Goblin,
　you should find that it does not become paralyzed.

　For information on how to teach the player character the skill "Paralyze",
　please refer to "I want to set the level at which the player character learns a technique".

　That concludes the content of this article. Good work!
　[Side Note]
■How status ailments are applied

　In this topic we configured resistance for the side receiving status ailments, but
　on the inflicting side, skills and weapons also have
　a probability of inflicting a status ailment condition.

　In the Basic System, the order in which these probabilities
　are calculated is: skill → (weapon) → resistance.

　The weapon's status ailment infliction rate is calculated separately from skills, so
　for example, if you equip a weapon that paralyzes with 100% chance and perform a normal attack
　against an enemy that is 100% susceptible to paralysis,
　even though normal attacks do not include a paralysis check,
　the weapon will still apply paralysis at 100%.

　Also, for the side receiving the status ailment attack, if allies' equipment or enemies
　have status ailment resistance, those resistances are subtracted from the infliction probability above.
■Setting resistance against element attacks

　In this topic we introduced how to configure status ailment resistance.
　For information on how to configure resistance against element attacks, please refer to
　"I want to set weakness, absorption, and immunity elements on enemies".
