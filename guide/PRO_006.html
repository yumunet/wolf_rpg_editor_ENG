【I want to get mouse input】

　[Goal]

　The best feature of mouse input is its intuitive operation.
　Once you can manage it well, you can even
　create games that run on mouse operation alone.

　WOLF RPG Editor is constantly monitoring the click state of the mouse.
　By calling the monitoring result into a variable, you can check at that moment
　whether a click is being pressed.

　This time, we will set up an event that talks when anywhere on the screen is left-clicked,
　and learn how to obtain mouse input.
　The contents of this event are:
　"Send a signal only while left-click is pressed,
　　and only launch the message display when that signal is present."
　By setting this event to Parallel Process,
　the click state is monitored at all times.

　【Steps】
[1] Create the event

　Create an event at any location in "Sample Map A",
　and set a chicken image for it.

　If you are unsure how, refer to steps [1]～[5] of
　"I want to create a new event".
[2] Trigger Condition

　First, change the Trigger Condition to "Parallel Process".
　The Trigger Condition is located just below where you set the event image.
　The "is left-click being pressed" check we are doing
　only checks at the moment it is called, so if you want continuous detection,
　the event must keep launching via Parallel Process.
[3] Detecting left-click

　Click "■ Show Command Input Window ■"
　at the bottom-right of the Event window.

　① Change the tab to "6 Variable Operation+".
　② For the variable type, select "Self" using the button.
　　 Choose "This Event: Self0" from the drop-down list,
　　 and select "=" as the assignment operator to its right.
　③ Next, move the radio button (the round button that places a black dot when clicked) to "Other",
　　 and from the drop-down list select "Mouse Left Click State (normal=0, pressed=1, newly pressed=2)".
　④ Press the "OK" button at the bottom-right of the window.

　The process of "sending a signal when the mouse left-click is pressed"
　is now complete.
[4] Conditional Branch and message display

　Now configure the settings so that a message is displayed when the signal is present.

　Change the tab to "7 Condition (Variable)" and select "Self"
　using the button for the variable type.
　In the first row of the three available Conditional Branch rows,
　specify "This Event: Self0" is (value) "1" "equal to".
　Make sure the checkbox for "Create an 'Otherwise' case"
　below the third row is unchecked.
　Once entered, press the "OK" button.
[5] Conditional Branch and message display

　Finally, insert a message command inside the branch just created.

　Place the cursor inside (the blank area of)
　"◇ Branch: [1] [If This Ev's Self Variable 0 equals 1] ↓",
　then from tab "Show Message",
　"OK" the message "You clicked!".

　The Event Commands that respond to clicks are now complete.
[6] Verify

　Let's do a Test Play.
　Click anywhere on the screen with the mouse,
　and if "You clicked!" is displayed, you succeeded!
　Great work!
　[Side Note]
· Creating clickable buttons
　Basic information retrieval commands such as mouse coordinates and click state
　are gathered under "Other" in command tab "6 Variable Operation+",
　but there is also "Is mouse cursor overlapping? (1=YES)" in the "Picture number" section.
　By combining picture display with this, you can create buttons that can be clicked and pressed.

· How to find the current screen coordinates of the mouse cursor
　System variables 71 and 72 always store the mouse's X and Y positions respectively.
　This information can also be obtained via Variable Operation+, but
　using these (system variables) saves some work.

· Moving messages and choices with the mouse
　System variable 73 contains
　a variable that turns "mouse-click input mode for messages and choices" on/off.
　When turned on, you can advance message display with a mouse click,
　or select and confirm a choice with a click.
　If you want it on, you need to toggle it via Variable Operation when the game launches, etc.
