【I want to get a string from keyboard input】


In WOLF RPG Editor you can directly input characters from the keyboard to overwrite a string variable.
Let's use this to change the player character's name during play.

[Goal]

　1. Create an event and get a string from keyboard input.

　2. Use a Common Event to change the player character's name.

【Steps】

―― [1] Create the event ――

　First, switch to "Sample Map A" using the Map Selector.
　Create an event in an easy-to-find location.
　Set the graphic to a chicken.
　(Reference: 【Event that launches with the Confirm key】)
　　

―― [2] Keyboard input ――　

　We will get a string from keyboard input.
　
　Click the
　"■ Show Command Input Window ■" button at the bottom-right of the Event window
　to open the Event Command Input Window.
　
　Select "5 String Operation" from the list on the left.
　
　The string assignment destination at the top-left remains "S0: [Sample] Temporary String".
　Click "Keyboard Input" near the middle.
　Then enter "14" in the "Character Count (half-width)" input field to its right.
　
　The string typed via the keyboard will now be assigned to
　"S0: [Sample] Temporary String".
　
　When done, click "OK" at the bottom-right of the window.
　

―― [3] Changing the name ――　

　Next, click "I Common Event" from the left list in the Event Command Input Window.
　
　Confirm that "Insert Event" is checked,
　then select "Common 6: ○ Change Player Character Info" from the drop-down list.
　Change the target player character to "12: Wolfar".
　
　When done, click "OK" at the bottom-right.
　
　(Reference: ◆ I want to change a character's name)
　

―― [4] Preventing the event from re-launching ――　

　Finally, as things stand, after pressing Enter in the keyboard input
　the "Execute on Confirm key" event will launch again.
　Let's add a wait to prevent this.
　(This is not necessary if a command with a built-in wait,
　　such as Show Message, follows this command.)
　
　Select "H Event Control" from the event input window.
　Click on "Wait" to enable it.
　1 frame is enough.
　
　Click "OK" at the bottom-right and it is complete.
　

　Save the map and do a Test Play.
　
　Talk to the event.
　When the input field appears, enter any name and press Enter.
　
　Open the menu and confirm that Wolfar's name has changed — if it has, you succeeded!
　Great work!

[Side Note]
　
　Numbers can also be entered via keyboard input.
　However, the input is assigned to a string variable,
　so it cannot be used directly as a numeric value.
　
　To convert it, write an expression like the following using Variable Operations and a variable-call value:
　
　"This Event: Self0 = S0: [Sample] Temporary String + 0"
　
　Assigning "3000000" to the right-hand side should produce this display.
　At that point, if the string variable's contents are half-width digits, they will be assigned to "Self0".
　
　However, reading stops at the first non-digit character,
　so if the first character is not a digit, the value cannot be read and will be treated as "0".


　For more details on variable-call values, see
　【◆ I want to specify a variable in the editor's value field】.
