【Is There a Way to Avoid Placing Similar Events Over and Over?】


　Events have something called a contact range.
　This time, let's use this feature to create the following type of event.

[Goal]

　1. Create an event that launches when touched.
　
　2. Expand the contact range of that event to cover a wide area with a single event.
　
　3. When the player enters the event's range, a Place Transfer occurs.
　
　

【Steps】

―― [1] Create Event ――

　First, use the Map Selector to switch to "Sample Map A".
　Create an event in the center of the raised platform at the bottom-left of the map (coordinate: 11, 20).
　
　We will not configure an event image this time.
　
　Referring to 【An Event That Launches When the Player Touches It】,
　set the event's Trigger Condition to Player Touch.
　

―― [2] Expand Contact Range ――　

　Expand the event's contact range.
　
　Look at the bottom-left of the event edit window.
　Set the "Contact Range Expansion" to 2 for both X and Y.
　

―― [3] Place Transfer ――

　Now let's configure the event content.
　
　Click the
　"■ Show Command Window ■" in the bottom-right of the Event window
　to open the Event Command Input window.
　
　
　Enter the Place Transfer command.
　
　Select "G Place Transfer" from the list on the left.
　The target should be "[ player character ]".
　If "Same Map" is checked, uncheck it,
　and set the destination map to [2: Sample Map B].
　
　Confirm that the "Precise Coordinate" checkbox on the right is not checked,
　and set the destination position to [X: 12 Y: 24].
　When done, click the "Input" button in the bottom-right.
　
　※ If anything is unclear,
　please refer to 【I want to move the player character to a different map】.
　

　The event is complete.
　Save the map and try a Test Play.
　
　Climb the ladder to the top of the raised platform (at the position shown in the figure on the right)
　and if the screen changes and you appear in Sample Map B, you've succeeded.
　Great work!

【Troubleshooting】

◆ Place Transfer does not occur when entering the raised platform
　The contact range expansion may not have been configured, or the event's Trigger Condition may be incorrect.
　
◆ Place Transfer occurred, but I didn't go to Sample Map B
　The Place Transfer command may have "Same Map" checked, or the destination map may be set incorrectly.
　Please check while referring to 【I want to move the player character to a different map】.
