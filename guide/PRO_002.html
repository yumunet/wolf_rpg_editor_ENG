【PRO_002: ◆Display JPG/PNG Images】

[Goal]

JPG and PNG images can be displayed as "pictures".
Once you can use pictures reasonably well,
you can display full-screen illustrations in events, create your own menus,
and if you master them fully, you can even make a shooting game
in what is supposed to be an RPG editor!

This time, we will perform the three most fundamental picture operations:
display, move, and erase — in that order —
to learn the basics of handling pictures.

【Steps】
0. Confirming terminology

To avoid confusion, let's clarify the difference between "image file" and "picture" before the steps.
"Image file" refers to "something in a folder (a file)",
while "picture" refers to "an image displayed in the game (loaded from an image file)".
Now, let's start with displaying a picture.

1. Displaying a Picture
Create an event at a suitable location in "Sample Map A"
and set the chicken image.

Click "■ Show Command Window ■"
at the bottom right of the Event window.
Change the tab to "A Picture".
There are many settings available, but
for now leave everything as-is and just set the file.
Click the "Load" button to the right of the "File" field
in the upper-center of the window.
The "Select Graphic" window will open.
Select "FaceGraphic_Yuichi.png" from the file list
and press the "OK" button at the bottom right of the window.
The image filename has been specified.
Next, enter X "160" and Y "120" in the "Coordinate" fields.
After entering, press the "Input" button at the bottom right.
The event content has been entered.
Let's do a Test Play.
If the chicken image is displayed around the center of the screen
after talking to the event, you've succeeded.

If it's "not displaying!", check the following points:

[1] Is opacity near 0?
"Opacity 0" = "completely transparent".
The maximum value of 255 is recommended for opacity.

[2] Is the zoom ratio near 0? (100 is recommended)
The closer the zoom ratio is to 0, the more the picture is shrunk,
and at exactly 0, it becomes completely invisible.
A zoom ratio of 100 is recommended, as it displays identically to the image file.

[3] Is the color 0, 0, 0?
When the display mode is not normal,
if all RGB color values are 0, it may not be visible.
100 is recommended as the base value for all three colors.
2. Moving a Picture
Before the command to move the picture,
insert a brief pause.
In the map Event window commands,
click the slot just below the picture display you just entered.

This time, we'll use a Show Message as the pause.
The content can be blank — enter the Show Message command.
The reason for inserting a pause is to make the change happen at a desired moment,
which makes it easier to observe.

(Actually, this Show Message pause is very useful when you want
to "stop and verify" while checking event behavior,
so it's worth knowing!)

After entering the Show Message command, click
the position just below that command.
We will add the "picture move" command at that position.
Let's move the position of the picture we displayed.
Open the Command Input window, change the tab to "A Picture",
and click the "Move" button at the upper right of the window.
Now configure the "display position" and "process time".

Enter "0" for coordinate X and "0" for Y as well.
(Confirm that "Relative Mode" and "Free Transform" checkboxes are unchecked.)

Enter "60" for process time and "0" for activation delay.

The picture move settings are now configured.
Press the "OK" button at the bottom of the window.
Let's do a Test Play.
When you talk to the event, the picture set in step 1 and
the message window should appear simultaneously.

Press the Confirm key to advance the message display and the picture should move.
If it moves to the upper left of the screen in about 1 second, you've succeeded.
If it doesn't move, review the steps.
3. Erasing a Picture
Finally, configure the settings to erase the picture.
Insert the same Show Message pause from step 2
again here.
Insert it just below the picture move command entered in step 2.

After inserting the pause, click the position
just below that to select it.
Open the Command Input window, change the tab to "A Picture",
and click the "Erase" button at the upper right of the window.
Enter "60" for process time and "0" for activation delay,
then click the "Input" button at the bottom right of the window.
When all commands have been entered,
it should look like this.
Let's do a Test Play.
The picture and message window should appear,
and pressing the Confirm key once should move the picture.
That covers steps 1 and 2.

Pressing the Confirm key again closes the
message window and simultaneously begins erasing the picture.

This time, the settings make it take about 1 second
from when erasing begins to when it is fully erased.
Settings are now complete. Great work!
4. About other settings

The picture command has many settings items, but
this time we only configured the display position and process time.
Here are some highlights from the other settings worth introducing.
If you have time, try configuring them yourself.

■Picture Number
Multiple pictures can be displayed on screen, but
"only one picture per picture number".
So to display multiple pictures, just use different picture numbers for each.
Picture numbers can be any integer up to 999999,
and when pictures overlap, the one with the higher number is displayed on top.
■Display Mode
In addition to "Normal", display modes include "Additive", "Subtractive", and "Multiplicative",
and these three are affected by the colors of what is displayed below.

"Additive" adds the picture's colors to the background (everything displayed below)
and displays them. Light adds colors, making them closer to white,
so additive display results in a brighter color than the background.

"Subtractive" is the opposite of additive — it subtracts the picture's colors from the background
and displays them. Reducing colors makes them closer to black,
so subtractive display results in a darker color than the background.
Since light information consists of three primary colors (red, green, blue),
subtracting a red picture from a white color leaves blue-green.

"Multiplicative" darkens the background by the darkness of the picture
(strictly speaking this is an approximation, but that's how it looks).
In additive and subtractive, white has the strongest effect,
but in multiplicative, the effect diminishes with brighter colors, and white changes nothing.
■Patterns
A picture can be divided by specifying the number of horizontal and vertical divisions,
and you can specify only one of the divided fragments (patterns) to display.
Patterns are arranged left-to-right in the image file,
and continue from the left of the next row after the rightmost.

For example, if both vertical and horizontal division counts are 3, and pattern 5 is specified,
only the center fragment of the 3×3 divided image will be displayed.
■Display Type
This time we displayed a picture by specifying a filename for file loading,
but there are other options:
displaying text as a picture, or automatically dividing an image
3×3 and displaying it like a window image.
These can be configured in the "Display Type" of the picture command —
try experimenting with them.
[Side Note]

So far we've been talking about Event Command "A Picture", but
finally let's briefly discuss what can be done with the "B Effect" command.

Here, you can change the color of an already-displayed picture,
make it flash, blink, shake, and more.
This can be quite useful when creating events.
For example, displaying a circle that overlaps a character and then
shaking or flickering it can easily produce a convincing aura-like effect.

Also, combining the "Pattern" feature mentioned in the side note above with
"Auto Pattern Switch" is enough by itself
to animate a picture.
Before entering, don't forget to confirm that the picture number is set to
the same number as the picture you want to apply the effect to.
