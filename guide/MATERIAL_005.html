【Use BGM/Sound Effect Assets in Your Game】

[Goal]
　While browsing websites, you may come across music-related assets.
　You may want to use music you like in the game you're creating.
　This article introduces how to use such music (BGM) and sound effects (SE) in your game.
　Here, we will create an event that changes the music when talked to,
　and an event that plays a specified sound effect when talked to.

★Note!
　When using assets, always read and comply with the asset's terms of use.
Also see "◆What are the rules for using assets?".

【Steps】

●Event that changes the music when talked to


[1] Where should assets be placed?　
　BGM assets should be placed in the "BGM" folder.
　This time, we'll use "Field01_Juno.mid".
　The BGM asset we'll use is already in the "BGM" folder.
The folder marked in red
[2] Register in the System Database!　
　Click to open the System Database.
　　①Register the BGM in an empty slot in System Database type 1 "BGM".
　　First, select an available data number, press the "File" button next to the filename field,
　　find the BGM you want in the dialog that appears, and click OK.
　　Also, enter "Field" in the ID field. This name will be referenced later.
Configure the red-outlined area.
The yellow-outlined area is explained in the section just below.
　　※About the other settings

　　"Volume (%)" is the volume at which the BGM will be played.
　　100 is the standard volume; values between 1–100 produce quieter sound, and above 100 is louder.
　　(Exceptionally, entering 0 plays the BGM at standard volume.)
　　Since unexpected volume levels may occur, be careful when configuring this.
　　
　　"Frequency (%)" is a value that changes the speed and pitch of the music by altering its frequency.
　　100 is the standard frequency; values between 1–100 produce slower music, and above 100 is faster.
　　(Exceptionally, entering 0 plays at standard frequency.)
　　For MP3, OGG, and WAV music assets, slower speed means lower pitch, and faster means higher pitch.
　　For MIDI music assets, only the speed changes — pitch does not change.
　　
　　"Loop Start (ms) / Key" has a meaning that differs depending on whether the BGM is MIDI or not, as noted in the Database comment.
　　If the registered BGM is MIDI, this is the key — the MIDI pitch shifts by the set value (changing by 1 shifts by a semitone; changing by 12 shifts by an octave).
　　If it's not MIDI, it is the loop start position — the BGM will loop from the time (in milliseconds = 1/1000 of a second) set here.
[3] Call it in an event!　
　①Create an event
　　Create a new event in Sample Map A. A chicken image is fine.
　　Once placed, double-click the event to open the Event Command window.

　②In the Sound command where you want to use it, select "Select Directly from System DB" and choose the BGM you want from the dropdown.
　　Here, find and select "Field" that we registered earlier.
　　Note that supported formats for BGM are .mid, .wav, .ogg, and .mp3 only.

　　※This command also allows you to configure "Fade Time" and "Resume Playback".
　　Setting Fade Time causes the BGM volume to gradually increase when it starts playing.
　　Resume Playback starts the BGM from the specified position — use this to play BGM from the middle.

　　※Specifying by filename when you need it
　　In the Sound command, select "Specify by Filename".
　　Pressing the "File" button opens a dialog from which you can select the BGM or other sound.
　　When specifying by filename, you can configure "Volume (%)", "Frequency (%)", and "Loop Position (ms) / Key" (BGM and BGS only) here.

　③Verify the behavior
　　Verify that the BGM changes by actually talking to the placed event.
　　Run Test Play and talk to the placed chicken.
　　If the BGM changes, you've succeeded.

　　Note that the music changes when you move to a different map because that map has BGM configured.
　　For how to set BGM for maps, see "◆Set Music for a Map".

●Event that plays a specified sound effect when talked to


[1] Where should assets be placed?　
　Sound effects should be placed in the "SE" folder.
　This time, we'll use [Effect]PUNYO_panop.ogg.
　This asset is also already in the "SE" folder.
The folder marked in blue
[2] Register in the System Database!　
　Click to open the System Database.
　①Register the sound effect in an empty slot of System Database type 3 "Sound Effects"
　　Just like with BGM, register the sound effect in the System Database.
　　Also, enter "Punyo" in the ID field. This name will be referenced later.
Configure the red-outlined area.
The yellow-outlined area is explained in the section just below.
　　※About the other settings

　　"Volume (%)" is the volume at which the SE will be played.
　　100 is the standard volume; values between 1–100 produce quieter sound, and above 100 is louder.
　　(Exceptionally, entering 0 plays the SE at standard volume.)
　　Since unexpected volume levels may occur, be careful when configuring this.

　　"Frequency (%)" is a value that changes the pitch by altering the SE's frequency.
　　100 is the standard frequency; values between 1–100 produce lower pitch and slower speed,
　　and above 100 produces higher pitch and faster speed.
　　(Exceptionally, entering 0 plays at standard frequency.)

　　Regarding the "Release from Memory" item:
　　This setting determines the loading method used by WOLF RPG Editor when playing this sound effect.
　　0 (Do not release) is suited for sound effects that play continuously,
　　1 (Release after playback) is suited for sound effects that play only once.
　　If you're unsure, leaving it at the default 0 is fine.
[3] Call it in an event!　
　①Create an event
　　Just as with BGM, create an event in Sample Map A. A chicken image is fine.
　　Once placed, double-click the event to open the Event Command window.

　②Configure SE in the command
　　In the Sound command, select "Select Directly from System DB" and choose the SE you want from the dropdown.
　　Here, find and select "Punyo" that we registered earlier.
　　Note that supported formats for sound effects are .wav, .ogg, and .mp3 only (.mid is not supported).

　　Note that the command also allows you to configure "Delay Playback" instead of "Fade Time".
　　For example, to delay a sound effect by 1 second, enter "60 frames".

　　※Specifying by filename when you need it
　　In the Sound command, select "Specify by Filename".
　　Pressing the "File" button opens a dialog from which you can select the BGM or other sound.
　　When specifying by filename, you can configure "Volume (%)", "Frequency (%)", and "Loop Position (ms) / Key" (BGM and BGS only) here.

　③Verify the behavior
　　Do a Test Play.
　　If the "Punyo" sound effect plays when you talk to this event, you've succeeded.
　　Note that sound effects do not loop.

[Side Note]

■How to stop BGM or other sounds
　In the Sound command, if you select "Select Directly from System DB", there is a "(Stop)" option.
　Selecting this will stop the BGM or sound effect.
　Note that setting a fade time allows the BGM volume to gradually decrease (fade out).
Set this to (Stop)
■What is BGS?
　First, the difference between BGM and BGS:
　These stand for "BackGround Music" and "BackGround Sound" respectively.
　One is background "music" and the other is background "sound".
　In Japanese, BGS is often called "ambient sound".
　Rain sounds, wind sounds, heartbeats, and similar things are well-suited for this (looping sound effects).
　By the way, sound effects are "Sound Effects = SE".

　The same methods described above apply to BGS as well.
　The database for registering BGS is System Database type 2 "BGS".
　Just like BGM and sound effects, there is a command to stop BGS.
　Supported formats for BGS are .wav, .ogg, and .mp3 only (.mid is not supported).
