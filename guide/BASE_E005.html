【I want to heal or deal damage to a player character】

[Goal]
　Make it so that touching an event causes the player to take damage
　or be healed.

　Bonus
　Version used: [2.x series]


[1]
　• First, use the Map Selector to switch to
　the "map where you want to create the event".

　• Create an event somewhere suitable
　and set it to the chicken image.

　For steps, refer to [1]–[5] of
　◆ An event launched by the Confirm key.




















[2]
　1. Once created, set the event's Trigger Condition
　to "Event Contact".

　2. From Event Commands, select
　I Common Event and confirm
　that "Call by Common Event Name" is selected.

　Since this is an event to
　"heal or deal damage to a player character",
　select "○ Status Increase/Decrease".

　3. For Target Player Character, choose a
　suitable character. It's best to specify
　a character that exists in the party during Test Play.

　For Which Stat, select "22: HP".

　For By How Many Points, enter "-10".
　　To deal damage, enter a negative value (-1 or less).
　　To heal, enter a positive value (1 or more).

　For Message, set it to
　"Yes" for now.




















[3] Before the Test Play

　1. Let's also add "Temporarily Erase This Event".
　As-is, the heal/damage process would trigger
　repeatedly; by erasing the event once it runs,
　we prevent it from running multiple times.


　2. If you want to run the process once without erasing
　the event, use a Condition (Variable) instead.

　Inside the branch, change the variable used in the
　condition so it no longer matches the condition.

　This way the heal/damage process will only
　run once.




















[4] Test Play
　Let's touch the event we created right away.
　Before touching it, open the menu and
　confirm that the character specified in
　[2] as the Target Player Character
　is in the party.

　If there is no such character in the party,
　go back to [2] and specify a character
　that is in the party.

　Now touch the event.
　If text is displayed as shown in the image
　when touched, you have succeeded.

　Great work! You're done.




















[5] Bonus
　For those who don't want the message window
　to stop the action every time it appears.
　Here is how to create the process using a picture to display the message,
　make it move, and then fade out while moving.




















[6] Process Setup Part 1

　Configure according to the image.
　1. Using 3 Variable Operation,
　add This Self Variable 0 = 0 + 0.
　※ This is the data position of the "Target Player Character".

　2. Using 3 Variable Operation,
　add This Self Variable 1 = -10 + 0.
　※ This is the "By How Many Points" value.

　3. In the red-underlined parts, specify the variables
　created in steps 1 and 2.

　To specify Self Variable 0 of the current event,
　enter "1100000".
　To specify Self Variable 1 of the current event,
　enter "1100001".





















[7] Process Setup Part 2

　1. Using Variable Operation, specify Self2 Absolute Value = Self1 + 0.
　　Absolute value calculates how far the value is from 0.
　　Since Self1 has -10 entered, the value 10 is assigned (=) to Self2.

　2. Call ▲ Get Member Info [String] to get the
　　target player character's name.
　　The target player character's name is assigned to the variable
　　specified in the "Result [Retrieved String]" entry below the red line.




















[8] Process Setup Part 3

　1. Branch using Self1 of this event.
　　Branch on two conditions:
　　Self1 is -1 or less, and
　　Self1 is 0 or more.

　2. Move into the branch where Self1 is -1 or less
　　and use 5 String Operation to assign (=) to S1:
　　" took \self[2] damage!"

　3. Move into the branch where Self1 is 0 or more
　　and use 5 String Operation to assign (=) to S1:
　　" recovered \self[2] HP!"




















[9] Picture display

　Enter values according to the image.
　For X, enter half of the screen width X.
　For Y, enter a value slightly less than the screen height Y.
　Pattern must be 1, and opacity 0.
　※ This is so the text gradually becomes
　　 opaque from a transparent state.
　Process time and activation delay are 0.
　Zoom and color are 100. Done.




















[10] Gradually display the text

　Enter values according to the image here too.
　Only the red-lined parts change.

　Select "Move".
　For Y, enter a value smaller than what was entered in [9].
　Set opacity to 255 and process time to 20.

　This makes the text slide upward from the bottom
　of the screen while gradually becoming opaque.






















[11] Erase after displaying for a certain time

　Enter values according to the image here too.
　Only the red-lined parts change.

　For Y, enter a value smaller than what was entered in [10].
　Set opacity to 0 and activation delay to 50.
　※ The activation delay makes [10]'s process
　　 start 30 frames after [9]'s process is
　　 fully complete.
　　 50 - 20 = 30




















[12] Cleanup
　1. Finally, select "Erase" for the picture
　and set process time to 0.
　Set activation delay to 70.

　2. Adjust so it only runs once, the same as [3], and you are done.

　The image shows the overall process.




















[13] Test Play
　Try changing the value of Self1 from -10 to various values
　to make sure it works correctly.


　If the text slides from the bottom of the screen upward,
　pauses briefly,
　then moves upward again while disappearing,

　and the specified character's HP has increased or decreased
　by the value of Self1, you have succeeded.
　※ However, HP will not exceed the maximum HP.

　To make the text slide for longer,
　increase the process time in [10] and [11].

　To extend the display time after the text pauses,
　increase the activation delay in [11]
　and also increase the activation delay in [12] by the same amount.

　This description is rough but we are done here.
　Great work! You're done.
