【An Event That Launches When Entering a Map】

[Goal]
　We will create an event that launches automatically the moment you enter a map.
　
　Here, we will create an event that displays the message
　"This is Sample Map B" when you enter Sample Map B.
　
　
　Something like this.

[1]
　1. Switch the map to "Sample Map B".
　　　
　　　・Press the Map Selector button.
　　　・From the "Map Selector" window that appears,
　　　　select "Sample Map B".
　　　
　
　2. Create a new event somewhere on the map.
　
　　　・Double-click the location on the map
　　　　where you want to create the event.
　　　・Alternatively, right-click the desired location and
　　　　select "Create/Edit Event".
[2]

　・We will not set a graphic this time.
　
　・Set the Trigger Condition to "Auto-execute".
　

　　"Auto-execute" is the key point of this tutorial.
　
　　"Auto-execute" refers to an event that
　　launches automatically
　　when you enter the map.
　　
　　※ For more details, please refer to the
　　　　[Side Note] at the bottom of this page.
　　
[3]
　Next, configure the event content.

　Click the
　"■ Show Command Window ■" button
　in the bottom-right of the Event window.
[4]
　・When the "Insert Event Command" window appears,
　　select "Show Message" from the tab at the top.

　・In the text input field,
　　enter 'This is Sample Map B'.

　・When done, click the "Input" button.
　

　※ We will be entering another command next,
　　　　so do not close the window yet.
[5]
　・Next, switch the tab at the top to
　　"Misc 1".
　
　・Select
　　"Temporarily Erase Event" from the lower part.

　・Set the target to "This Event" and
　　enter [0] for the fade.

　・Once configured, click the "Input" button. 


　The command configured here
　temporarily erases the event.
　Without this, the message will
　keep displaying endlessly.

　That's all the commands to enter.
　Close the window.
[6]
　Let's confirm.
　The Event Command display should show…… 　
　■ Message: This is Sample Map B
　■ Temporarily Erase Event: This Ev (0 frames)
　■
　
　Does it look like this?
　
　
　Now let's do a Test Play.
　
　Click "Save (Entire Map)", then
　click "Test Play".
[7]
　・After the opening demo ends,
　　keep moving upward to
　　move to Sample Map B.
　　
　・When you move there, you should immediately see
　　"This is Sample Map B"
　　displayed just once.
　　
　・Go back to the previous map and then
　　return to Sample Map B again,
　　and it should display in the same way.
　　

　　This completes the tutorial.
　　Great work!
　　
【Troubleshooting】
　
　・The message isn't displayed!
　　→ Please confirm that the Trigger Condition is set to "Auto-execute".
　　
　・The message keeps displaying repeatedly and I can't move!
　　→ Check that the command
　　　■ Temporarily Erase Event: This Ev (0 frames)
　　　has been entered correctly.
　　　Without the Temporarily Erase Event command, it will repeat endlessly.

[Side Note] Details on Auto-execute Events
　
　An Auto-execute event is an event that starts automatically when the player character enters the map where it is placed.
　
　These are often used for staging/effects.
　For example, hearing a scream upon entering a cave, or a scene playing out in a boss chamber...
　
　Important points to keep in mind:
　
　・The player character cannot be controlled while it is running, and other events cannot be launched either.
　
　・Auto-execute events continue repeating as long as their conditions are met.
　　Therefore, the event must be removed as soon as it finishes.
　　The following methods are generally considered:
　　
　　1: Remove it with "Temporarily Erase Event"
 　　　　Just like the event created in this tutorial, it will display again if you leave and return to the map.
　　
　　2: Use variables and the Event Trigger Condition to permanently erase the event
 　　By using self variables or similar to switch event pages, you can prevent the event from launching,
　　so it won't run even if you leave and return to the map.
　　→ Related: ◆An Event That Disappears Once and Never Returns
