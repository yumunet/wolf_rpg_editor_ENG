【I want to set a character's step to half-tile or 1-tile units】

　[Goal]

　A character's step unit can be set to full-tile (1 step = 1 tile)
　or half-tile (1 step = half a tile, 2 steps = 1 tile).
　Full-tile movement games make setting tile passability very easy,
　while half-tile movement games allow finer expression than full-tile settings —
　each has its own advantages.

　This time, we will learn how to switch
　a character's step unit between half-tile and full-tile.

　【Steps】
[1] Open the Settings screen

　Select "Open Game Basic Settings" from "Game Settings",
　and open the "Game Basic Settings" window.
[2] Change the setting

　You can configure this under the item "Default Character Movement Width".
　Select either 0.5 tile or 1 tile with the radio button,
　then press the "OK" button at the bottom-right of the window.
　The settings are now done.
[3] Test Play

　Let's do a Test Play.
　With full-tile settings, you can only walk on tiles that are fully passable,
　so for example the warm bed in the building at the lower-right of Sample Map A
　can no longer be stepped on.
　Great work!
　[Side Note]
　■ Advantages and disadvantages of full-tile vs. half-tile

· The greatest advantage of full-tile movement settings is that configuration
　is many times easier compared to half-tile movement.
　This is recommended for those who are not yet experienced with game creation.
　However, full-tile movement means you can't stand at 0.5-tile positions,
　so you can't walk exactly through the center of a symmetrical map,
　and roads that were passable in a half-tile map will become impassable
　if full-tile settings are applied to them.
· The strength of half-tile movement settings is that it allows more precise configuration,
　and you can create passages that wouldn't be accessible with full-tile settings.
　It handles 0.5-tile precision well and enables fine-grained behavior
　in both map creation and Location Transfers using half-tiles.
　However, the settings are correspondingly more demanding — unexpected shortcuts may be discovered,
　and testing passable areas in the map can take more time.
　Also, the fine movement may cause the character to get caught on impassable tiles,
　which could be frustrating depending on the player.
· When making a game, the best time to decide which step unit to use is
　at the very beginning of development,
　before you start creating maps.
　Changing it later means you will need to review all maps and map events created so far,
　which is a lot of work.
　■ Default collision detection

· Inside the "Default Character Movement Width" section of Basic Settings,
　there is a "Default Collision Detection" setting.
　This "collision detection" determines the area in which
　the player character is considered to be touching events or map tiles,
　and it can be configured when the default character movement width is 0.5 tile.
　When the movement width is 1 tile, the collision detection is unconditionally "1×1 tile",
　but when the movement width is half-tile, you can choose between "1×0.5 tile" and "1×1 tile".
　This is width × height, so in either setting
　the player character has 1 tile of collision detection in the horizontal direction.
· What changes with this setting is the size of the vertical collision detection.
　Try each setting and stand right next to a wall or event directly above or below the player character —
　you can tell the difference by how close the player character approaches.
　With 1×1 tile, the player character stops at a position where there appears to be
　a half-tile gap between the character's shadow and the object (wall or event),
　while with 0.5 tile, there is almost no gap between the shadow and the object.
