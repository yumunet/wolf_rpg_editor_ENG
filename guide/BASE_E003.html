【I want to increase/decrease held items】


[Goal]
• Make it so examining the chicken gives the player an item
• Prevent the player from receiving the item a second time
• After obtaining the item, delete the chicken graphic

For details on obtaining items from treasure chests, see ◆ A treasure chest that stays open once opened.


[1]
• First, use the Map Selector to switch to the "map where you want to create the event".

• Create an event somewhere suitable and set it to the chicken image.

For steps, refer to [1]–[5] of ◆ An event launched by the Confirm key.


[2]
Once created, click "■ Show Command Input Window ■" at the bottom right of the Event window.

1. When the "Insert Event Command" window appears,
　click the Other 2 tab. (Image 1)

2. Confirm that "Insert Event" (circled in the image) is selected. (Image 2)
　If the circle next to the text has a dot in it, it is selected.
　※ If it is not selected, click it to select it.

3. Clicking the area circled by box 3 will display a list
　of available Common Events.
　(Image 3) Since this is an item acquisition event,
　select "Common 0: ○ Item Increase/Decrease".


[3]
Next, edit the "Common Event Input (Value)" field.

1. "Item Number" selects the
　item to obtain.
　※ You do not need to worry about
　the "[User 2]" notation at the end.
　
2. "Increase/Decrease Amount" sets
　how many of item 1 to increase or decrease.
　Setting a negative value will
　decrease the item count.

3. "Acquisition Message" controls whether
　the message "Obtained ○ × ○" is displayed.
　You can also skip it here and display
　it later with "Show Message".
　
4. Finally, press the "Input" button (arrow).

The process for obtaining an item is now complete.


[4]
The item acquisition event is complete up to this point,
but as-is the player will be able to obtain the item every time they interact.
(Click here if that is fine.)

Let's create a process to prevent that.
Don't worry too much about what the process does — just configure it as described in [5] below.

For more details, see ◆ A treasure chest that stays open once opened.


[5]
• Find and click "Variable Operation" in the "Insert Common Event window".

• Confirm the parts highlighted with red lines. Configure them to match the image on the right.
　
• Finally press Input at the bottom right.

What was configured here is a process that assigns "1+0=1" to "Self Variable 0 of this event".


[6]
The image shows the top-left of the Event window.
Pay attention to the part enclosed in yellow.

1. Click the "New Page" button.

2. Click the "Page 2" button to open Page 2.


[7]
A new page with no graphic or event has been created.
Page 2 is the state of the event after the item has been obtained.
Set the graphic to "No Image" (default).


[8]
Let's configure the "Event Trigger Condition".
Pay attention to the part enclosed in purple.
　
1. Click the leftmost checkbox to add a checkmark.

2. Click the "Self" button to select it.

3. Select "Self Variable 0".

4. Enter the value "1".

5. Select "Same as".

The event content stays empty.

This completes Page 2: an event with no image and nothing happens when examined.


【Save】
Save the map and do a Test Play!
Clicking the Test Play button in the Event window will save automatically.


【Test Play 1】
First, let's examine the event.

Successfully obtained the item!
The message is displayed correctly too.

※ If something goes wrong...
• Check if the Item Number is wrong.
• Check if Increase/Decrease Amount is 0 or negative.
• Check if Acquisition Message is set to "0: None".


【Test Play 2】
Press the Cancel key to open the menu and check the Items section.

You can also confirm that the event image (circled behind the window) has disappeared.

※ If something goes wrong...
• Check that you used the right Self Variable in Variable Operation.
• Check that you haven't overwritten that Self Variable.
• Check that the assigned value is correct.
• Check that you used the right Page.
• Check that the Trigger Condition on Page 2 is correct.
　(Refer to [10] in ◆ A treasure chest that stays open once opened.)

That was a long one, but good work!
This concludes "I want to increase/decrease held items".
