【I want to disable saving from the menu screen】

　[Goal]

　A game where you can save anytime, anywhere gives a sense of security,
　but there is actually a special effect to the mechanic of
　only being able to save at designated save points.

　For example, imagine the "Save" command is named "Report",
　and during the story the villain steals your report book,
　temporarily preventing you from saving until you get it back.
　What kind of effect would that create?

　You can create a tension that the story alone cannot express.
　That's right — a sense of security is not the only thing that makes a game good.

　Also, in a game where you can save anywhere, if a "dead end" (a state where
　the game can no longer progress, sometimes called a "soft lock") occurs and
　the player saves in that state, all their previous play becomes wasted.

　By applying save restrictions, you can prevent the player from saving
　in a dead-end state and resolve this problem.

　All of this can be done with "disabling saving from the menu".
　This time, we will learn how to create a save-restricted map by making Sample Map B save-restricted.

　(The save point mentioned above is covered in
　 "I want to open the menu or save screen from within an event"
　 for those who need it.)

　【Steps】
　This time we will restrict saving only on Sample Map B,
　so we will do two things:
　"Disable saving when moving from Sample Map A to Sample Map B" and
　"Allow saving when moving from Sample Map B to Sample Map A".
　■ Disabling Save
[1] Open the Event

　We will configure saving to be disabled when moving from Sample Map A to Sample Map B.

　Double-click to open event "06: [Location Transfer to B]" on "Sample Map A".
[2] Select Where to Insert the Save-Disable Command

　The right portion of the window is occupied by the Event Command field.
　We will insert the save disable command immediately after the Location Transfer command already there.

　Click once on the command just below the Location Transfer command,
　"Run Transition", to place the cursor there.
[3] Enter the Event Command

　Click "■ Show Command Input Window ■"
　at the bottom right of the Event window.

　① Change the tab in the Event Command Input window to "I Common Event".
　② Select "Insert Event" using the radio button
　　 (this is selected by default).
　③ From the dropdown list on the right, select "Common 15: ○ Save Allow/Disable Settings".
　④ In the [Common Event Input (Value)] field,
　　 set "Mode" to "0: Disable Saving".
　⑤ Finally, press the "Input" button at the bottom right of the window.
[4] Confirm the Event Command Insertion Position

　Confirm that the command has been inserted in the correct position.
　Close the Event Command Input window and check the Event Command field
　in the Map Event window.
　It is OK if "Insert Event" appears between Location Transfer and Run Transition.
　■ Allowing Save
[1] Open the Event

　Next, we will configure saving to be allowed when moving from Sample Map B to Sample Map A.

　Double-click to open event "02: Location Transfer to A" on "Sample Map B".
[2] Insert the Save Command

　From here, follow the same steps as Section 1 to
　input the Save Allow/Disable Settings Common Event.
　However, this time set "Mode" to "1: Allow Saving".
　■ Confirming Behavior
　Do a Test Play to confirm that saving is correctly disabled/allowed.
　When saving is disabled, the "Save" option in the menu should appear slightly darker
　and cannot be selected even when you press the Confirm key.

　First confirm that saving from the menu works on Sample Map A,
　then confirm that saving is disabled on Sample Map B.
　If saving becomes available again after returning to Sample Map A, you're all set.
　If any one of these isn't working correctly, please review the steps.
　Great work!
　[Side Note]
■ Disabling saving for an entire dungeon

　This time we disabled saving on only 1 map, but if you want to disable saving
　for an entire dungeon, you only need to set save disable/allow at the
　dungeon entrance's Location Transfer.
　Since the save disable/allow setting is basically not changed by anything other than this command,
　if you set it to "Disable Saving" upon entering the dungeon, it will remain disabled
　no matter how many Location Transfers occur — until you set it to "Allow".

　By the way, a save-restricted dungeon (depending on enemy/trap difficulty)
　gets harder as the dungeon grows larger, so consider placing
　save points and heal points that restore HP and other stats.
　You could place a rest area in the middle of the dungeon, or right before the boss — whatever works.

　How to create save points and heal points is covered in separate topics,
　so please refer to those.
　• Save point → "I want to open the menu or save screen from within an event"
　• Heal point → "I want to heal or deal damage to a player character"
■ Using "save disable" as a way to prevent soft locks

　Events from which you cannot return carry the risk of a "soft lock".
　A "soft lock" refers to a state where the game can no longer progress,
　and in some cases it is one of the most serious bugs a game can have.
　In this topic we will use the term "soft lock" to describe this state.

　Common examples of soft locks include:
　　• Being cornered in an unwinnable forced battle with no way to go back,
　　　 leaving you with no option but Game Over.
　　• Moving pushable rocks on a map in the wrong order,
　　　 making it impossible to advance or retreat.
　　• Entering a dungeon you can't exit until cleared, finding the enemies too strong,
　　　 wanting to level up and try again, but being unable to leave due to the design.

　If you simply soft-lock during play, you normally just reset and try again more carefully,
　but saving while in a soft-locked state is the worst case scenario.
　Resetting won't fix it, so you're forced to start over from the beginning.
　After all the time the player spent getting that far, losing their motivation to continue
　would be such a waste...

　If you find an event or situation during Test Play that might cause a soft lock,
　it might be worth considering adding "save disable" as a first step to resolving it.

　　• Disable saving before entering an unwinnable forced battle.
　　• Disable saving before entering a map with pushable rock puzzles.
　　• Disable saving before entering a dungeon you cannot leave until cleared.

　There may be other effective solutions in some cases, but "save disable" can
　often resolve soft locks surprisingly cleanly.

　However, when using "save disable", always pair it with "save allow".
　Whenever you disable saving somewhere, make sure to set "save allow" once that event ends.

　If there are multiple exit paths (be especially careful of Location Transfer spells, etc.),
　you must set "save allow" on every exit path.
