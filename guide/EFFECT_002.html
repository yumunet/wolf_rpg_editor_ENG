【I want to darken the screen or make it flash】

[Goal]

One way to create presentation effects in games is to change the screen brightness.
This time we will introduce how to darken the screen (fade to black) and how to make it flash.
There are various methods, but here we will use "Screen Color Tone Change".
This method affects all elements displayed on the screen, including maps, events, and pictures.

Darkening the screen is often used when you want to change the scene in the story.
In this tutorial we use the method to go completely black, but by changing the RGB values described later, you can also create a "slightly darkened" state,
or even make the "screen turn completely red" (not easy on the eyes, but could work for horror effects...).
Flashing is used in the Basic System when enemies appear and battle begins, for example.
It could also be used for scenes like taking a photo, or a key item's gem activating its power.

【Steps】

◆ Darkening the screen

[1] First, create an event on the map.

Let's make it so that when you talk to this event, the screen darkens.
The familiar chicken
[2] Create the event contents

In the Event Commands, select "effect".
We will use "Screen Color Tone Change" from there.

Let's set each of R, G, B to 0.
(Clicking the "Go to Black" button will immediately set all RGB values to 0.)
Set the processing time to 30 frames. The screen will darken over approximately 0.5 seconds.
Set everything to zero, as shown.
[3] Add a process to restore the screen, thinking ahead

This operation will darken the screen, but unless you do something, the brightness will never return.
This time, let's also add a process to restore the screen.

Select "Event Control", set "Wait" to "60" frames, and press the "input" button.
(This pauses the process for about 1 second. See "◆ I want to add a wait time" for more on Wait.)
Then, in "effect", use "Screen Color Tone Change" and set each RGB value to 100.
(Clicking the "Color Reset" button will immediately set all RGB values to 100.)
Set the processing time to 30 frames. The screen will return to normal over approximately 0.5 seconds.
Clean up after yourself — don't leave things in a broken state (laughs).
[4] Let's try it

Talk to the chicken we placed.
If the screen darkens and then returns to its original brightness, it worked.

For reference, the sequence of events this time is:
"The screen goes completely dark over 0.5 seconds, then after 0.5 seconds pass, the screen brightness returns over another 0.5 seconds."
*Note

If you don't include the process described in [3], the screen brightness will not return.
However, key inputs will still be accepted normally, so the player can operate the game while the screen is completely black.
This can lead to unexpected results, so when you make the screen go black, it is a good idea to stop processing with a Wait or similar.
Also, don't forget to always include a process to reliably restore the color tone afterwards.

One more note: if you set a value to "200", the entire game screen will become that single color and you won't be able to see anything.
In that case, reduce the value to adjust.

◆ Flashing the screen

[1] First, create an event

Place an event the same way as in the screen-darkening example.
Our familiar chicken again
[2] Select "Screen Color Tone Change" and prepare for flash

Set all R, G, B values to 200. This will make the screen turn white.
(Clicking the "Go to White" button will immediately set all RGB values to 200.)
This time, check the "Make Flash" checkbox. This will create the flash effect.
Let's try a processing time of 30 frames. This will result in approximately 0.5 seconds of flash.
Don't forget to enable the flash.
[3] Let's try it

If talking to the chicken causes the screen to flash white, it worked.
After the flash, the screen brightness should return to normal.

[Side Note]

● About RGB
RGB represents the "Red", "Green", and "Blue" color components of the screen respectively.
With "Screen Color Tone Change", you can set values between 0 and 200.
The larger the value, the stronger that component appears. If it is 0, that component does not appear at all, so setting everything to 0 makes the screen completely black.
Note that the initial value is 100, where each component appears at its original strength.
● Things you can do with RGB values
R=130, G=130, B=130: the screen turns whitish. (Setting values to 200 will make it completely white and invisible.)
R=100, G=0, B=0: the screen turns reddish. Good for horror effects... (Same note applies. For flash, values of 200 are also OK.)
R=50, G=50, B=70: the screen darkens (with a slight blue tint). Good for night effects, etc.
...Try various other combinations as well.
Top-left (100,100,100) Top-right (100,0,0)
Bottom-left (130,130,130) Bottom-right (50,50,70)
● About processing time
There is a processing time setting; for color tone changes, the screen color changes over that number of frames.
For flash, it works slightly differently.
The entire sequence — from the screen changing to the set color until it returns to the original tone — takes the number of frames set as the processing time.
During this, the time for the initial "screen changing to the set color" phase becomes longer as the processing time increases.
It takes roughly 1/4 to 1/3 of the processing time in frames.
Note that entering "0" for the processing time will cause the flash not to occur.
