【I want to detect whether a specific key has been pressed】


By using the Key Input command, you can check whether a specific key has been pressed.
This time we will create an event that checks which of the three keys — Confirm, Cancel, or Sub-key — was pressed.

[Goal]

　1. After talking to the event, accept a key input.
　
　2. Display a message corresponding to the key that was pressed.

　■ Related topics
　【I want to detect whether multiple keys are pressed simultaneously】

【Steps】

―― [1] Create the event ――

　First, switch to "Sample Map A" using the Map Selector.
　Create an event in an easy-to-find location.
　Set the graphic to a chicken.
　(Reference: 【Event that launches with the Confirm key】)

―― [2] Accepting key input ――　

　Click the
　"■ Show Command Input Window ■" button at the bottom-right of the Event window
　to open the Event Command Input Window.
　
　Click and select "9 Key Input" from the list on the left.
　Leave the assignment destination and key type as they are,
　then click the direction key drop-down and select "Do Not Accept".
　
　Check all four checkboxes below that.
　
　When done, click OK at the bottom-right of the window.
　
　※ If "Wait until key is pressed" is checked,
　　 the process pauses at the Key Input command
　　 and subsequent processing will not run until the configured key is pressed.
　　

―― [3] Judging the input (Conditional Branch) ――　

　With the settings in [2], when a key is pressed the result is assigned to [This Ev: Self 0].
　Based on that value you can determine which key was pressed,
　or whether no key was pressed at all.
　
　Now select "7 Condition (Variable)" from the left of the window and configure it.
　
　We will use three branches, so check three condition checkboxes on the left.
　Set the values from top to bottom to "10", "11", "12" (equal to),
　then click "OK" at the bottom-right.
　

―― [4] Create the messages for each branch ――　

　Now build the contents of each branch.
　
　Look at the Event window.
　Start with the case where "This Ev Self Variable 0 equals 10",
　click on it to move the cursor there.
　
　Select "1 Show Message" from the left of the Command Input Window.
　Enter "The Confirm key was pressed."
　
　Similarly, when the value is 11 it means the "Cancel key" was pressed,
　and when it is 12 it means the "Sub-key" was pressed.
　Enter messages for each.
　

　This is now complete.
　Save the map and do a Test Play.
　
　First, talk to the event.
　At this point only your movement is stopped and nothing else changes, but
　try pressing one of the keys: Confirm, Cancel, or Sub-key.
　
　If the message corresponding to the key you pressed is displayed, you succeeded!
　Great work.
　

【If a problem occurs】

◆ Nothing was displayed and the character could still move.
　The event's Trigger Condition is wrong.
　Or "Wait until key is pressed" may not be checked.
　Please verify.
　

[Side Note]

· Notes for when not waiting for a key press

　When not using the "Wait until key is pressed" feature, a little care is needed.
　
　If there is a wait of 2 or more frames inside a Loop or parallel event that contains the command,
　the input state cannot be checked every frame.
　In other words, there will be moments when pressing a key produces no reaction.
　
　In that case, either prepare a dedicated parallel event,
　or set up an environment that can check input every frame by using event calls, etc.


· How to handle multiple key inputs simultaneously

　If you try to accept multiple keys with a single Key Input command,
　priority rules will overwrite results so that only one input can be obtained at a time.
　
　However, by using multiple Key Input commands with different assignment destination variables,
　you can obtain multiple key inputs simultaneously.
　This can be used to detect simultaneous key presses.
