【How Should I Create Map Chip Assets?】

【Question】
Map chips appear frequently on screen when creating a normal game.
Map chips also play an important role in presenting the world of the game.
So, what kind of specifications should map chips used in WOLF RPG Editor be made to?

For simplicity, this description uses a screen size of 320×240. The chip size is 16 pixels square.
For other sizes, please see the 【Appendix】.

【Solution】

■ First, the image format
A 256-color transparent PNG is recommended.

■ Map chip size
Width: 128 pixels × Height: a multiple of 16 (N in the figure) pixels

■ Auto-tile size
Width: (16 × number of animation patterns (P in the figure)) pixels × Height: 80 pixels
This is what a map chip looks like; N is the number of rows and can be any value.

This is what an auto-tile looks like; P is the number of patterns.
【Details】
■ For image format, a PNG with 256 colors and a transparent color is best.
(See "◆ I Want to Make Images Transparent" for transparency.)
If a transparent color is not set here, the image will not display correctly in the edit screen,
and the game screen will also not display as intended.
■ Map chips are basically arranged with 8 chips in a row horizontally.
You can then extend it vertically as much as you like.

As an example, the base map used in the sample is 128×3984 pixels wide and tall.
The chips are arranged in 8 columns and 249 rows.
This is what the sample chip looks like.
■ Auto-tiles are arranged as shown in this image.
From top to bottom:
Red: "This auto-tile has none of its kind on any of the four sides (appears at the outer corners of a right-angle placement)"
Yellow: "This auto-tile has its own kind above and below"
Green: "This auto-tile has its own kind to the left and right"
Blue: "Appears at the inner corners when this auto-tile is placed at right angles"
Purple: "Completely surrounded on all sides by this auto-tile"
These are the patterns.
Arranging all patterns gives the layout shown in the figure.
Example-use patterns have also been provided. We hope this helps you get an image of how it works.

For additional animation, simply keep adding more patterns to the right.
In the actual game screen, the display switches to the next pattern at regular intervals, then cycles back to the first pattern.
These are all the available patterns.

One example — compare it with the patterns above.
As an example, the waterfall chip in the sample has 4 animation patterns.
The size is 64×80 pixels wide and tall.
In the game screen, you should be able to see the water flowing animation.
This is an image also in the sample.
Arranged horizontally like this, it animates.
★ Please pay attention to size.
Even 1 pixel off will cause an error like the one described in "◆ I Got an Error Saying the Map Chip Must Be 128 or 256 Wide".

【Appendix】

★ For other game screen sizes
■ At 640×480, the chip size is 32 pixels square.
Map chip: width 256 pixels × height a multiple of 32 pixels
Auto-tile: width (32 × number of animation patterns) pixels × height 160 pixels
■ At 800×600, the chip size is 40 pixels square.
Map chip: width 320 pixels × height a multiple of 40 pixels
Auto-tile: width (40 × number of animation patterns) pixels × height 200 pixels
That is how it works.
★ Environment-dependent notes
In some environments, when the screen size is set to 640×480,
there are reports that auto-tiles with 1 animation pattern (width 32 × height 160) are not displayed.
This appears to be a bug where images below a certain size are not recognized.
The solution is to add the same pattern image again horizontally to increase the width to 64 pixels.
