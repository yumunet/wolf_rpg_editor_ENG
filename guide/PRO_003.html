【I want to specify a variable in the editor's value field】

【Overview】
In WOLF RPG Editor, large integers of 1,000,000 or more are not just plain numbers —
some of them specify variables.
By entering these values in an Event Command's value field,
you can call a specific variable in their place.

For example, if you enter the large integer "1103000000" — which specifies "money held" —
in a value field, 1103000000 is automatically replaced with "money held" when processed.

So what can you do with this feature?
Let me describe it concretely using the following two examples.

① An event that doubles the money held
┗Enter the variable-call number in the editor's value field and...

② Making the bread you eat on-site at the bakery cost money
┗Try calculating using the variable-call value.

① An event that doubles the money held

"(money held) × 2" can also be expressed as "(money held) + (money held)".
Adding the current amount of "money held" to money held doubles the money held.

In this event, we call the variable value (amount) of "current money held",
and add it to money held using an Event Command.
The key point here is the method for calling the "current money held" amount in [2] below.

Let's try it.
[1] Create a new event, set its graphic,
open the "Command Input Window", and switch the tab to "Other 2".

Reference: ◆ I want to create a new event [1]～[7]
[2] Select "Common 3: ○ Increase/Decrease Money".
Here is the key point.
In the "Money Change Value" field, enter exactly "1103000000".
On the surface this number appears to be 1.103 billion, but as a variable it specifies "money held" —
more precisely, "Changeable DB: Type 3 Party Info: Data 0 Main Settings: Item 0 Money Held".

After entering it, look at the area highlighted in red at the bottom of the image on the right.
It should display:
"1103000000 = [Changeable DB] Type 3: Party Info
　　　Data 0: Main Settings / Item 0: Money Held (variable)"
(If it says something different, please verify that the value you entered is correct.)

This is the name of the variable being called.
Variable-call values have many digits and it is easy to make mistakes,
but even a single wrong or missing digit means the value
cannot be retrieved correctly. So verification here is important.
[3] Once verification is done, press "OK".
Next, to confirm the event ran, enter a message.
Switch the tab to "Show Message", type "Money doubled!" and press "OK".
[4] Now let's do a Test Play.
At the start of the game the money held is 0, so obtain some money first
and then trigger the event. Each time you trigger the event,
the money held will keep doubling.

You should now understand that what is added to the money held each time the event fires
is not the "value" of 1.103 billion, but the variable "money held".

That was a demonstration of what happens when you enter a variable-call value in the editor's value field.

Next, let's use another example to perform a calculation using a variable-call value.

② Making the bread you eat on-site at the bakery cost money

This is the event where eating bread lets you use a group attack.
Eating on the spot is currently free, but let's add a process to this event that
"deducts the bread's price from money held" to make it cost money.

This time, in [3] we will use a variable-call value to retrieve
the price of the item "Delicious Bread" from User Database item data.
[1] Open [Event:9 - Cook] and select the message choice [1]\s[0]
(the choice with the line "Alright, just wait a moment!")
and open the Command Input Window, then switch the tab to "DB Operation".
[2] First, configure the upper half — the DB settings — to set "money held".
Select Changeable DB, enter "3" for the DB type number, "0" for the data number, and "0" for the item number.
There is a verification field at the bottom of the window, so check there that the settings are correct.
If correct, it should display:
"▼ Changeable DB Type 3 Party Info / Data 0 Main Settings / Item 0 Money Held".
[3] Now we use the variable-call value.
This time it is "1002000319", which represents the bread's price.
Enter this in the "Value to Process or Assign Destination" field in the middle of the window.

If entered correctly, the verification field should display:
"▽ 1002000319 = [User DB] Type 2: Items
　Data 3: Delicious Bread / Item 19: Sale Price [0 = Cannot Sell] (variable)"
[4] We have just retrieved money held from the Changeable DB and the bread's price from the User DB.
Now let's subtract these two variables.

Since we are subtracting the bread's price from money held, change the "="
immediately to the left of where you entered 1002000319 to "−=".

The option further to the left — "Assign to DB ↑" or "Assign to Variable →" —
should be set to "Assign to DB ↑".

The process of "deducting the bread's price from money held" is now finally complete.
[5] Now for Test Play. Note that the baker does not appear until
the event that adds P.G. Eddy to your party has occurred.

Did the money held decrease by 20 Gold?
If it didn't work, read through steps [2][3][4] above again while looking at WOLF RPG Editor.
If you find a mistake, fix it; if you can't find one, try re-entering the values.
Trial and error without giving up is what builds your knowledge.

By the way, even if your money held was less than 20 Gold, it will still subtract 20 Gold,
so the money held will become a negative value and the event will proceed anyway.
We won't do it here, but if you have the time, try modifying it so it works properly.
Please keep at it until you get it. Great work!
[Side Note]
Variable-call values are complex large integers, so you may be thinking "Do I have to memorize all of these?!"
— but you don't need to memorize them.
You can always view them in WOLF RPG Editor under Help → "Variable Call Values / Special Character List".
Besides, if you use them frequently you will eventually memorize them,
so it's not as difficult as it may seem at first.

Variable-call values are a useful feature. However, they cannot be used in just any text field.
Variable-call values cannot be used in places where non-numeric input is expected,
such as message text or string variables (use special characters there instead), but as a general rule,
any place where entering a number displays "Normal integer value" in the verification field
will accept variable-call value input.
