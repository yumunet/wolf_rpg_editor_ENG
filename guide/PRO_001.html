【PRO_001: ◆Learn How to Use Parallel Events and Auto Events in Detail】

[Goal]

Among event Trigger Conditions, Auto-execute and Parallel Process have the characteristic that
"they are executed when the condition is met, and continue running until the condition changes".

Here, we will refer to events with an Auto-execute start condition as "auto events",
and events with a Parallel Process start condition as "parallel events".

When an auto event is running, no other trigger condition events
(Execute on Confirm key, Player Touch, Event Touch, Auto-execute) can launch except "parallel events",
but when a parallel event is running, all trigger condition events including "parallel events" can
launch — this is the key difference.

By understanding how to use auto events and parallel events,
you can insert scene events into your game or set up gimmicks during exploration events.
They are also unavoidable when creating your own combat system or message display system.

This time, we will create a function "that starts a timer when entering the dungeon and teleports the player outside when time is up",
and use it to learn the basic usage of auto events and parallel events.

【Steps】
[1] What is needed for the function we will create

For this event creation, we will use 2 regular variables,
a new event "Timer Event" with 3 pages,
and the existing event "Entrance Location Transfer".

■■■Regular Variables■■■
■Event Management Flag A variable that manages the state of the three events. Set as follows:
  • 0: Timer is not running.
    When entering the dungeon in this state, the timer start event launches.
  • 1: Timer is running.
    In this state, the timer variable increases.
  • 2: Timer has reached the specified value.
    When this state is reached, the timer stops and the timer exit event launches.
■Timer Variable A variable that increases by 1 each frame (1/60 second by default).
When this value reaches 1800 (30 seconds), the "Event Management Flag" is set to state 2 (timer exit).

■■■Timer Event■■■
The main timer process event placed inside the dungeon.
■Page 1: Timer Start Event
An Auto-execute event that notifies the player that the timer has started.
After displaying a message, it starts the timer.
■Page 2: Timer Process Event
A Parallel Process event that increases the timer variable.
After increasing the timer variable, it checks whether the value has reached 1800 (30 seconds) or more,
and determines whether to call the timer exit event.
■Page 3: Timer Exit Event
An Auto-execute event that notifies the player that time is up.
After displaying a message, it teleports the player character outside the dungeon
and resets the timer run state (= Event Management Flag) back to the initial state (= 0).

■■■Entrance Location Transfer■■■
An existing movement event from the dungeon to Sample Map A.
■Page 1: Move to Sample Map A
Save is enabled and the command to move to Sample Map A is already entered.
This time, we will add a timer reset process here for when the player exits the dungeon midway through the timer.
[2] Creating the function
■Creating the regular variables
From the System Database, open type 14: "Regular Variable Names".
Set ID: 1 to "Event Management Flag" and ID: 2 to "Timer Variable".

■Creating the Timer Event
Create a new event at coordinate (0,0) (top-left) of Map 3 "Dungeon".
Set the event name to "Timer Event" and set the graphic to "None".
■Page 1: Creating the Timer Start Event
(1) Setting the event Trigger Condition
Confirm that the event page is "Page 1".
Set the Trigger Condition to "Auto-execute",
and configure it to run when "v1: Event Management Flag" is "equal to" "0".
(2) Picture display
Using the command "A Picture", set picture number 1,
display picture as text, with text "\v[2]".

※This displays the value of the timer variable on screen.
(3) Show Message
①Using the command "1 Show Message", configure the following message:

-----
※The timer is starting.
It seems Wolfar and friends cannot
stay in this dungeon for more than 30 seconds.
-----

※This displays a message notifying the player that the timer has started.
(4) Variable Operation: V1 [Event Management Flag]
①Configure the Variable Operation as follows:

V1: Event Management Flag, =, 1

※Sets the Event Management Flag to 1 to start the timer.
(5) Variable Operation: V2 [Timer Variable]
①Configure the Variable Operation as follows:

V2: Timer Variable, =, 0

※Sets the timer variable to 0, initializing it to the starting state.
■Page 2: Creating the Timer Process Event
(1) Setting the event Trigger Condition
Confirm that the event page is "Page 2".
Set the Trigger Condition to "Parallel Process",
and configure it to run when "v1: Event Management Flag" is "equal to" "1".
(2) Variable Operation: V2 [Timer Variable]
①Configure the Variable Operation as follows:

V2: Timer Variable, +=, 1

※Each time the event is called, the timer variable value increases by 1.
(3) Picture display
①Configure the following picture display settings:

Picture number 1, display picture as text, text "\v[2]"

※Each time the event is called, the timer variable value is re-displayed.
(4) Conditional Branch (variable)
①Configure the Conditional Branch as follows:

"V2: Timer Variable" is "1800" "or more"

※Checks the timer variable value to determine whether to perform the flag change process.
Visualize a number line and set the condition naturally.
(5) Variable Operation: V1 [Event Management Flag]
①Inside the Conditional Branch configured above, set up the Variable Operation as follows:

V1: Event Management Flag, =, 2

※Sets the Event Management Flag to 2 to exit the timer.
■Page 3: Creating the Timer Exit Event
(1) Setting the event Trigger Condition
Confirm that the event page is "Page 3".
Set the Trigger Condition to "Auto-execute",
and configure it to run when "v1: Event Management Flag" is "equal to" "2".
(2) Show Message
①Configure the following message:

-----
※The timer has ended.
30 seconds passed, and Wolfar and friends
were sent flying to the dungeon entrance.
-----

※This displays a message notifying the player that the timer has ended.
(3) Picture erase
①Configure the following picture settings:

Picture number 1

※Erases the picture displaying the timer variable.
(4) Location Transfer
①Configure the Location Transfer as follows:

Target: player character
Destination map: 1, Sample Map A
Destination position: X: 34, Y: 12
Transfer option: Perform transition [with fade-out, slow]

※Moves the player to the dungeon entrance.
(5) Variable Operation: V1 [Event Management Flag]
①Configure the Variable Operation as follows:

V1: Event Management Flag, =, 0

※Sets the Event Management Flag to 0, resetting it to the initial state.
(6) Variable Operation: V2 [Timer Variable]
①Configure the Variable Operation as follows:

V2: Timer Variable, =, 0

※Sets the timer variable to 0, resetting it to the initial state.

■Modifying the Entrance Location Transfer
(1) Picture erase
Configure the following picture settings:

Picture number 1

※Erases the picture displaying the timer variable.
(2) Variable Operation: V1 [Event Management Flag]
Configure the Variable Operation as follows:

V1: Event Management Flag, =, 0

※Sets the Event Management Flag to 0, resetting it to the initial state.
(3) Variable Operation: V2 [Timer Variable]
Configure the Variable Operation as follows:

V2: Timer Variable, =, 0

※Sets the timer variable to 0, resetting it to the initial state.

■Verifying the behavior
1. Do a Test Play to verify the behavior.
Key points to verify:

(1) Timer start
- A timer start message is displayed immediately upon moving to the dungeon.

(2) Timer process
- The "Timer Variable" value is displayed in the upper left and increases.

(3) Timer exit
- When the number in the upper left reaches 1800, a timer exit message is displayed,
and the player character moves to the dungeon entrance.

If something goes wrong, re-check the settings content.
Trace through the flow of the process one step at a time, and set it calmly.
[Side Note]
Is it really exactly 30 seconds?

Actually, it may not be exactly 30 seconds.
By changing the title bar information of the game, you can check how many FPS the game is actually running at.
FPS (Frames Per Second) is the value indicating how many frames are processed per second.

To put it roughly,
WOLF RPG Editor is running an extremely fast flip-book animation (!).
The parallel process is triggered once per frame of this flip-book (once per 1/60 second),
and the setup of this event is: count the timer variable (+1) each time it is called, and raise the flag when it reaches 1800.

If you run many parallel events at once, depending on what they do,
the expected frame rate may not be achieved and it may take longer than 30 seconds,
but for a small-scale parallel event like this one, you generally don't need to worry about it.

What to watch out for while the Timer Event is running?

Whenever there is a process spanning multiple events, you need to consider the exit conditions,
and this situation is no different.

If the player moves to another location midway through the timer event,
without a process to exit the event midway, the timer variable's picture display may remain at the destination,
or the value may remain without being reset.
If you are creating a warp spell that lets you return to town freely, take special note.

In this sample game's case,
if a timer reset process is not added to the existing event "Entrance Location Transfer",
bugs will occur.
