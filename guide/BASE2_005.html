【Is the "defend" effect during combat also a status ailment?】

　【Question】

　Status ailment data is stored in User Database entry 8 "8: Status Settings",
　but in addition to negative statuses like poison and paralysis,
　it also contains status ailments intended for positive use,
　such as defend, max HP increase, and attack boost.

　Are these statuses with positive effects also considered "status ailments"?

　【Explanation】
The term "status ailment" carries a strongly negative image, but
as the name implies any abnormal state (including defend and power charge),
everything from unusual states all the way to incapacitation counts as a status ailment.

Both negative and positive effects are applied to the character receiving the skill,
so for example, if you attack an enemy with a weapon that inflicts the "Attack Boost" state,
the enemy will receive the Attack Boost state.

Status condition settings can be configured in User Database entry 8
"8: Status Settings". Let us look at the most commonly used settings in 【Details】.
　【Details】
《Page 1》

　① Minimum Duration [Turns] [-1: Permanent]

　Also known as the "base turn count", this is the
　baseline number of turns the ailment lasts. The entry below,
　"Duration Variance [±]", can make it longer or shorter than this value.
　Entering -1 creates a status ailment that does not
　heal from turn passage alone. Entering 0 creates a
　status ailment that heals on the same turn it is applied.


　② Duration Variance [±]

　This entry adds randomness to the number of turns the status ailment lasts,
　and applies when the entry above "Minimum Duration [-1: Permanent]" is 0 or more.


　③ When this status is stacked ↓

　This entry is configured in conjunction with the entry below "┗ Next state to enter".
　If configured to cure when stacked with incapacitation,
　it becomes a status ailment that can avoid incapacitation once
　(though HP is not healed, so the character is near death).


　④ ┗ Next state to enter

　This entry is configured as a pair with the entry above "When this status is stacked".
　In the sample game, these two entries are used to give
　the "Power Charge" status multiple stages.


　⑤ Action Restriction

　This entry configures action restrictions, such as being unable to act
　while paralyzed or asleep, or being unable to choose targets and actions
　while confused.

　There is also a feature that forces allies (or enemies) to use specific commands,
　so with creativity you can make status ailments like "Vampire" that continuously
　absorbs allies' HP, or "Berserk" that makes the target blindly rush at enemies
　and take recoil damage.


　⑥ Special Skill Restriction

　This is a status ailment that puts the target in a sealed state
　where magic use is sealed and cannot be used.
　For status ailments other than the sealed state,
　keep this set to "[0] No Restriction".
《Page 2》

　・HP Change ±[%] / End of Turn
　・HP Change ±[Fixed] / End of Turn
　・SP Change ±[%] / End of Turn
　・SP Change ±[Fixed] / End of Turn

　These 4 entries can be configured for status ailments like the poison state where HP gradually decreases,
　or what some RPGs call "Regenerate", where
　the value gradually recovers each turn.
　What gradually heals or decreases with these entries is not max HP/SP,
　but current HP/SP.


　・Enemy Targeting Rate

　Setting this to "[1] Focused Front Attacks" makes it so that even when receiving
　an all-target attack, only the character with this status ailment is hit.
　"[-1] Not Targeted [※Still Hits]" and "[-2] Not Targeted & Full Immunity"
　have the effects as described.
　In the sample game, this setting is used for "Hide" and "Taunt".
