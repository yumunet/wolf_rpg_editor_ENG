【An Event That Runs in Parallel with Regular Events】

[Goal]
When you set an event's Trigger Condition to "Parallel Process", those events (unless a variable is specified as a trigger condition)
will keep running in parallel with other events at all times.
(For map events: while the player character is on that map; for Common Events: they keep running on any map)
Let's learn how to use "Parallel Process" so you can start using it right away.


※ Regarding the difference from the previous section ◆An Event That Launches When Entering a Map, i.e., "Auto-execute"

An "Auto-execute" event, while running, prevents all other events except Parallel Process from operating.
On the other hand, a "Parallel Process" event keeps running regardless of whether other events are launched or not.
[1]
As an example, let's create a clock.
First, as usual, create an event with a chicken appearance. An event image is not actually necessary for this process (and is rather in the way), but it serves as a marker.

※ This time it will not say "Cock-a-doodle-doo".
[2]
Next, let's configure the process content. After setting the "Trigger Condition" to "Parallel Process", enter the Event Commands.
First, from "H Event Control", add "Wait 1 Frame". This is apparently a charm to prevent bugs. Since it has an effect, make sure to add it.
[3]
Next, select "A Picture". There are many options, but the number of settings items for this tutorial is not large.



First, set the "Picture Number" at the top to "10".
※ Other numbers are fine, but avoid conflicting with other pictures in the sample game.

Next, from "Display Type", select "[3] Draw string as a picture". This is what the Professor mentioned.
A string input field will appear inside the "Display Type" frame.

Here, we'll use special characters to read "system variables" in order to display a clock.
Enter "\sys[80]:\sys[81]:\sys[82]".

The colons are to make it look like a clock, but the important part is "\sys[]".
This is a special character that means "display the value of system variable number x here (x is the number inside [])".
System variable 80 retrieves the current [hour], 81 retrieves [minute], and 82 retrieves [second].
WOLF RPG Editor handles this automatically, so simply entering the special characters will display the current time.

Click "Input" at the bottom-right to finish.


[3]
Now save the map and test the event you created. Click "Save (Entire Map)", then click "Test Play".

[4]
Did the clock appear in the upper-left of the screen? If strange characters are displayed instead of the time, please confirm that you entered the special characters correctly.


With a map event, the time will only display on the map where it is placed. If you want the time displayed on all maps, create a "Common Event".
Since there is no Variable Operation or similar in this process, simply set the Trigger Condition to "Parallel Process (Always)" and enter the same commands.

This completes "An Event That Runs in Parallel with Regular Events". Great work!


If you're curious about "system variables", take a look at the System Variable/String List in the official manual (the list is near the very bottom of the table of contents).
There are so many that it can feel overwhelming.
Come back and look at it later when you think "I wish I could get the value of this position..." or "I want to do this kind of process...".
