【◆ There is a MIDI that does not play when used in WOLF RPG Editor】

　　　[MIDI asset sound does not play]

[Goal]
　Get a MIDI file that produces no sound (partially or entirely) to play correctly.
　
　As a sample, we have prepared a MIDI that plays normally in regular music players
　but loses its melody in WOLF RPG Editor — please use it for testing.
　
　sample.mid (right-click → Save target as file)
　
　※Note※
　Depending on the situation, you may need to modify the MIDI asset itself.
　Be sure to read the terms of use for the downloaded asset, and take great care to avoid any issues.

[1]
　※ Since this procedure may produce unexpected sounds,
　it is recommended to remove headphones and listen through speakers.
　
　First, check your audio environment.
　Right-click the volume icon in the notification area (bottom right of the desktop)
　and open the Volume Control.
　
　Check each item to make sure the volume is not set to 0,
　and that the mute checkbox is unchecked for everything except "Input Monitor".
　
　Then listen to the MIDI asset once in this state.
　If there are missing sounds compared to listening with your usual music player (such as Windows Media Player),
　the asset uses sound tones not supported by WOLF RPG Editor, so proceed to [2].


[2]
　Sounds missing on the WOLF RPG Editor side can be temporarily fixed
　by modifying the contents of the asset.
　Since this involves modifying the asset,
　you will need software for MIDI editing.
　
　The following software requires no complicated settings
　and is used here for this explanation;
　we will modify the asset so it plays correctly.
　
　Cherry　MIDI Sequencer
　http://www.vector.co.jp/soft/win95/art/se071842.html
　（Vector）
　
　Download and extract it,
　set up the folder, and launch cherry.exe.
　
　Window descriptions:
　Main window: file saving, software settings, song playback, etc.
　Track List window: a window displaying overall information about the song
　Track #1～ windows: windows for editing individual parts
[3]
　Select "Open" from "File"
　and load the MIDI file you want to edit.
　When loading is complete, the sound data will be displayed
　as red lines in the "Track List" window — confirm this.
　(For the sample, CH2, CH3, CH4, and CH10 contain
　sound information.)
[4]
　Let's check the instrument information.
　Display the Track #1～ window and select CH2,
　which contains sound data, from the box at the top left.
　
　Data will appear below;
　find "Program" among them and double-click it.

[5] A screen for selecting a sound tone will appear as shown on the right;
set the MAP number to "000" and the BANK number to "000".
Press OK when done. (Leave PC# unchanged.)
Now this part (CH2) uses a sound tone compatible with WOLF RPG Editor.
[6] If you know which sound tone part is not playing, you only need to fix that one (in the sample, only CH2),
but if you are not sure, repeat [4]～[5] for all parts except the percussion part (CH10).
(For the sample, apply settings to the 3 parts: CH2, CH3, and CH4.)

Afterwards, confirm how it sounds in WOLF RPG Editor and you are done.
