【An error saying "The maximum map coordinate is..." appeared when transferring】

【Situation】
The "Location Transfer" command is used to move from one map to another map.
However, you may encounter an error like "The maximum map coordinate is...".
This time, we will describe how to resolve this issue.
For Location Transfer, see "◆ I want to move the player character to a different map".
This is a fundamental skill — be sure to learn it.

Specifically, this error occurs when the destination coordinate in the Location Transfer command exceeds the size of that map.
For example, the "Sample Map A" map size is "40 wide, 30 tall". The maximum coordinate is therefore "39, 29".
If you specify the destination coordinate as "40, 30" in the Location Transfer command,
that is outside the map boundaries and cannot be transferred to, causing an error.
Note that when the error occurs, the destination is forcibly set to a position that fits within the map.
This is the kind of error you'll see.

【Solution】
Set the destination so it fits within the map size.
Click the wrench icon to bring up the map settings, and check the size of the destination map.
The size minus 1 is the maximum coordinate you can transfer to on that map.
By specifying a coordinate that falls within that range, you can avoid the error.
Here it is.
◆ A more reliable prevention method
When setting up a Location Transfer, using "Specify while viewing destination" lets you
display the destination map and specify the transfer point from there,
ensuring you can always pick a coordinate that fits within the map size.
Here it is — the convenience of this feature is invaluable...

[Side Note]

◆ Location Transfer: then and now
Up to Ver1.11, there was no "Specify Location Transfer while viewing map" feature.
You simply specified the "destination map" and "destination coordinate" to perform the transfer.
As a result, this error probably occurred much more frequently than it does now.

Even from Ver1.12 onwards, when specifying the destination using a variable, this error can still occur.
If an error does appear, review the part of the code that performs this operation.
There may be an unexpected value stored in the variable.
For this case, you have no choice but to use the map size (which can be obtained with Variable Operation+)
and ensure the value stored in the variable does not exceed the transferable range.
It depends on the situation, so deal with it as needed.

Another case is when you change the size of the destination map but forget to update
the destination in the Location Transfer event.
When you change a map's size, go back and review the related events once.
