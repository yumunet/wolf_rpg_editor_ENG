【I Want to Make Images Transparent】

[Goal]

For images used in games, character chips and map chips typically have transparency processing applied.
Without this, the issue shown in the image on the right will occur.
Windows' built-in Paint and similar tools cannot handle transparency information. A dedicated tool of some kind is needed.
Here, we will describe the process using "EDGE", a free pixel art editor, as an example.
What we will be doing is designating one color used in the image as the transparent color (making areas painted with that color transparent).

For detailed operating instructions, please refer to the manual included with EDGE.
By the way, the EDGE developer's site is here (TAKABO SOFT). There is an EDGE section within it.
Make sure this doesn't happen to you (this uses a different image from the chicken on the left)
＊Note!
EDGE can only handle images with up to 256 colors (full-color images are not supported).
If an error occurs that prevents you from opening a file in EDGE, please use a color-reduction tool.
As an officially recommended color-reduction tool, there is "OptimumBMPtoPNG" and others.
If you get this kind of error, please reduce the colors by some method.

【Steps】

This time, let's use an image we've prepared here and make the black-painted areas transparent.
We will apply transparency processing to this chicken so it doesn't look unnatural when used in the game.

First, save this image to your computer.
If you are using "Internet Explorer", you can right-click the image and select "Save image as" to save it.
The image we will use this time.
[1] Prepare the image. Is it a 256-color image?
　Open the image you want to make transparent in EDGE.
　Full-color images cannot be opened.
　Note that the image being used this time has already been reduced to 256 colors.
Open it.
[2] Which color do you want to make transparent?
　Select the color you want to make transparent in the image.
　In EDGE, the color palette screen (see figure) lists all colors registered in that image.
　In the bottom-right of the palette screen, next to the color number display,
　the left side shows the drawing color (the color used when drawing), and the right side shows the background color (the color used for transparency in this case).
　Confirm that the color you want to make transparent is selected as the background color.
　If a different color is selected, right-click the color you want to make transparent on the palette
　and confirm that the desired color is now selected.
　Note that right-clicking on the image itself (when a tool like pencil or line is selected)
　changes the drawing color to the color at the cursor's current position.
　You can use that as a basis to select the background color from the palette.
Check the number in the red box! In this image, this color is used as the background.
You cannot make other colors transparent!
[3] Make the image transparent and save
　Run "Save As" and check "Transparent Background Color" at that time (choose .png for the format).
　Saving this way will save an image with the background color made transparent.
Don't forget this!
[4] Confirm
　Actually use it in the game and confirm via Test Play that the transparency has been applied.
　If the area specified as the transparent color has been properly made transparent, you're good to go.
　Sometimes it may appear non-transparent in the editor (when a color other than black is used as the transparent color), but it will be correctly transparent in the game screen.
It's properly transparent now.

[Side Note]

  • The "tkool2WOLF+" described in the article "◆ I Want to Use RPG Maker-Sized Assets in WOLF RPG Editor" also has a transparency feature.
    For details, please see that article.

  • Also, if you want to draw an image from scratch and apply transparency, for 256-color images, it's a good idea to reserve color number 0 as black (RGB = (0, 0, 0)) as the transparent color.
    Of course, when saving, select palette number 0 as the background color before doing "Transparent Background Color".
    If you need to use black in the drawing, prepare black at a different palette number, or use a black with a slightly different RGB value.

  • For image formats, transparency is supported by png. (GIF also supports transparency, but cannot be used in WOLF RPG Editor.)
    BMP (bitmap) and JPG cannot handle transparency information.
