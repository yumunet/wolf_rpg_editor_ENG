【What Are the Rules for Using Assets?】

　【The Question】

　Game development requires not only the creativity to write a scenario
　and programming skills, but also files such as BGM and face graphics.
　In principle, all of these should be made by yourself.
　However, making all the images, music, and other files for your game
　yourself is a lot of work.
　That's where you need the help of people skilled in art or music.
　Borrowing "assets" for your game is a very important choice.

　Assets refer to image files such as character chips and face graphics,
　audio files such as BGM and sound effects,
　and script-type assets such as Common Events —
　specifically, those for which the creator has permitted others to use them.
　In other words, images, music, or Common Events whose authors have not given permission to use in games are not assets.
　However, just because something says "you may use it" does not mean
　you can handle it however you like — there are basic rules,
　as well as terms of use specific to each creator or asset.

　The strictness of the terms of use varies by degree —
　from assets where you don't need to credit them in your readme,
　to assets where readme credit is mandatory, modification is prohibited, and usage reports are required.
　There is truly a wide variety of terms of use.

　This article summarizes the most common general rules for using assets on the internet.
　The purpose of this topic is to reduce problems when using assets.
　What is written here represents general principles only.
　When actually using assets, follow the terms of use of each individual asset.

　【Explanation】
■General asset usage etiquette

　Music and images on websites can be
　borrowed and used if the creator has expressed their intention to allow use (= terms of use).
　As a general rule for handling borrowed assets, it is best to follow:
　"Do not modify (no modification)"
　"Do not give to others (no redistribution)".
　Also, "when distributing your work, clearly state in the readme the asset creator you borrowed from"
　will minimize problems.
■About modification

- "Modification" refers to using dedicated software (image or audio editing software) to
　alter part of a file.

- For image files, this includes changing colors or moving parts.
　For audio files, it includes transposing or changing the timbre.

- In WOLF RPG Editor, for example, playing an MP3 file at a different frequency,
　or displaying only part of an image file —
　these are not using dedicated software to edit the asset itself,
　so most asset creators do not consider this modification.
　(Stricter creators may care about this too,
　so if you're unsure, re-check the terms of use.)

- In general, modification should be avoided, but
　map chips and similar assets are still scarce for WOLF RPG Editor.
　That's where the convenient tool "tkool2WOLF" comes in,
　which converts assets made for RPG Maker format to WOLF RPG Editor format — but this
　does count as "using editing software to change part of the file",
　i.e., it is definitely "modification".

　Therefore, first check the asset's terms of use for "whether modification is allowed"
　and "whether use in WOLF RPG Editor is permitted".
　If modification and use are permitted, the method of using tkool2WOLF for conversion is
　described in "Use RPG Maker-Sized Assets in WOLF RPG Editor".
■About the prohibition of redistribution

- "Redistribution" means giving the asset itself to someone else (transferring it).
　Whether it is paid (sold) or free is irrelevant.
　Anyone other than the person who created the asset giving it to someone else is redistribution.
　Being told not to do this is the "prohibition of redistribution".

- Also, when distributing your completed game, if you leave asset files unencrypted,
　they can easily be extracted.
　There are various opinions on this, but some asset creators
　do not like it.
　Encrypting assets makes it hard to extract them easily, so
　when distributing or publishing your game, it is desirable to encrypt using WOLF RPG Editor's encryption feature.

- However, note that some files may stop working after encryption
　(font files, for example, cannot be loaded by WOLF RPG Editor after encryption),
　so checking and noting whether things work after encryption is necessary.
■About credits and usage reports

- "When distributing your work, clearly state in the readme the asset creator you borrowed from"
　means writing it clearly in a place that can always be read.
　For example, the following format is recommended:

- Create a separate Readme.txt from the game itself, add a section like "Assets",
　and write the asset creator's name, the name of the website where the asset is distributed,
　and the website URL. It is common to use the top page URL.
　If the asset was modified, adding a comment like "used with minor modifications"
　is even better.

- Some asset creators require usage reports,
　so when obtaining assets, create a folder for each creator
　and keep a note in a text file so you don't forget.
　Being able to properly manage assets and showing respect to those you borrowed from —
　both are necessary mindsets for game developers.
　[Side Note]
　There are a few more important things about asset usage,
　so let's cover them here.
■About RTP

- RPG Maker's RTP is an asset only permitted for use in RPG Maker series software,
　so neither the original assets nor modified versions of them can be used in WOLF RPG Editor.
　Even if you own and have purchased an RPG Maker series product,
　you cannot use it in WOLF RPG Editor, so please take note.
- Using "RPG Maker VX" assets as an example:
　"VXRTP" and "VX-modified" assets cannot be used.
　Original assets merely created in VX format can be used.
- It's easy to understand when something clearly states "for use in RPG Maker only", but
　if it's not clearly stated, you need to be careful.
　Confirm that the asset you want to use "is not an RPG Maker RTP derivative" before
　borrowing it.
　If you're unsure, it is wiser not to use it.
■I found a file I want to borrow — where can I tell if it's an asset?

- Generally, the terms of use for an asset will be displayed prominently
　near the file's distribution page.
　Information about whether it can be used in games should also be written there,
　so checking for keywords such as "may be used in original tools" or "created for WOLF RPG Editor"
　gives you peace of mind.
- Even items on an asset creator's homepage cannot be borrowed
　if they are not assets (e.g., background images on the page).
■Summary

- As stated at the beginning, making all the images, music, and other assets for your game yourself is a lot of work.
- Each individual asset was made with effort by its creator,
　so when borrowing, the most important thing is to
　be respectful and not cause any unpleasant feelings.
- For those new to borrowing assets,
　following the stance of:
　"Do not modify (no modification)"
　"Do not give to others (no redistribution)"
　"When distributing your work, clearly state in the readme the asset creators you borrowed from"
　is recommended to avoid problems.
　Also, being willing to simply not use things you're unsure about is necessary.
- As you become more experienced with game development, you may ask asset creators
　to make assets for you, or make requests.
　Even then, never forget that creating assets is hard work.
