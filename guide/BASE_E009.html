【I want events to react differently based on held items, money, or party members】

　[Goal]

　When you want to create an event like "a gatekeeper who opens the door if you give 1000G",
　if you try to deduct 1000G from the player's money without checking whether they have enough,
　you'll end up with a situation where the money goes negative but the player is still let through —
　which is clearly unnatural.
　For events like this, you can create the ideal event by checking "whether the player has 1000 or more gold"
　and branching based on the result.

　In this topic, we will learn how to check held items, held money, and whether a specific character
　is in the party, and create events that react differently based on those results.
　This teaches you a simple way to check quantities and member presence, as well as how to branch.

　This time we will create three events: "an event that reacts to whether the player has the dungeon key",
　"an event that reacts to how much money the player has for staying at the inn", and
　"an event whose reaction changes if P.G. Eddie is in the party".

　【Steps】
■ Event that reacts to whether the player has a key

　We will create an event that displays "I have the key." if the player has the dungeon key,
　or "I don't have the key." if they don't.
[1] Create the Event

　Create an event at a suitable spot in "Sample Map A" and
　set it to the chicken image.

　If you are unsure how, refer to steps [1]–[5] of
　"I want to try creating a new event".
[2] Enter the Event Command: Get Item Count

　Click "■ Show Command Input Window ■"
　at the bottom right of the Event window.

　① Change the tab to "I Common Event" and configure the event operation process fields.
　② Select "Insert Event" using the radio button
　　　(this is selected by default).
　③ From the dropdown list on the right, select "Common 17: ▲ Get Item Count".
　④ In the [Common Event Input (Value)] field, set "Item" to "19: Dungeon Key".
　　Then set "→ Result [Item Count]" at the bottom of the window to "This Ev: Self0".
　⑤ Finally, press the "Input" button at the bottom right of the window.

　This creates a process that retrieves the number of keys held and
　stores that information in a variable (like a drawer).
[3] Enter the Event Command: Enter the Conditional Branch

　Next, we'll access the variable (drawer) we just stored data in,
　and create the part that reacts differently based on the information inside (the number of keys held).
　
　① Change the tab to "7 Condition (Variable)".
　② For the variable type, select Self.
　③ In the dropdown list directly below, select "This Event: Self0".
　④ Set the value field on the right to 1, and the condition to "Or More".
　⑤ Check the checkbox at the bottom left: "Create Else Branch".
　⑥ Finally, press the "Input" button at the bottom right of the window.
　
　This creates a branch that separates "has a key (= holds 1 or more)" from
　"otherwise (= 0 or fewer)".
[4] Enter the Event Command: Enter the Branch Content

　Now enter the content for each branch.
　
　Close the Command Input window, and enter the content for the key-count branch you just created.
　Using Show Message, enter
　"I have the key." inside "◇Branch: [1][ This Ev's Self Variable 0 is 1 or more ]↓", and
　"I don't have the key." inside "◇Else".
　
　If you are unsure how to enter messages, refer to [7][8] of
　"I want to try creating a new event".
[5] Confirm the Event

　Does the event content look like the image on the right?

　You have now created an event that says "I have the key." if the player holds even one key,
　or "I don't have the key." if they have none.
　Do a Test Play and confirm that it actually works.
　Great work!

■ Event whose reaction changes if P.G. Eddie is in the party

　We will create an event that displays "It's an old man!" if P.G. Eddie is in the party,
　or "It's a lady!" if he is not.
[1] Create the Event

　Create an event at a suitable spot in "Sample Map A" and
　set it to the chicken image.

　If you are unsure how, refer to steps [1]–[5] of
　"I want to try creating a new event".
[2] Enter the Event Command: Get Member Info

　Click "■ Show Command Input Window ■"
　at the bottom right of the Event window.

　① Change the tab to "I Common Event" and configure the event operation process fields.
　② Select "Insert Event" using the radio button
　　　(this is selected by default).
　③ From the dropdown list on the right, select "Common 22: ▲ Get Member Info [Value]".
　④ In the [Common Event Input (Value)] field,
　　　set "Target Player Character" to "13: P.G. Eddie [Var 0]", and
　　　set "Info to Get" to "-1: Present (1) / Absent (0)".
　⑤ Set "→ Result [Retrieved Value]" at the bottom of the window to "This Ev: Self0".
　⑥ Finally, press the "Input" button at the bottom right.

　This creates a process that retrieves whether Eddie is in the party and
　stores that information in a variable.
[3] Enter the Event Command: Enter the Conditional Branch

　Next, we'll refer to the variable we just stored data in,
　and create the part that reacts differently based on the information inside (whether Eddie is present).
　
　① Change the tab to "7 Condition (Variable)".
　② For the variable type, select Self.
　③ In the dropdown list directly below, select "This Event: Self0".
　④ Set the value field on the right to 1, and the condition to "Same As".
　⑤ Check the checkbox at the bottom left: "Create Else Branch".
　⑥ Finally, press the "Input" button at the bottom right of the window.
　
　This creates a branch that separates "P.G. Eddie is present" from "otherwise (not present)".
[4] Enter the Event Command: Enter the Branch Content

　Now enter the content for each branch.
　
　Close the Command Input window, and enter the content for the branch you just created.
　Using Show Message, enter
　"It's an old man!" inside "◇Branch: [1][ This Ev's Self Variable 0 is Same As 1 ]↓", and
　"It's a lady!" inside "◇Else".
　
　If you are unsure how to enter messages, refer to [7][8] of
　"I want to try creating a new event".
[5] Confirm the Event

　Does the event content look like the image on the right?

　You have now created an event that says "It's an old man!" if P.G. Eddie is in the party,
　or "It's a lady!" if he is not.
　Do a Test Play and confirm that it actually works.
　Great work!
■ Other things you can check

　This time we introduced methods using Common Events to check
　"held item count", "held money", and "specific party member presence",
　but there are many other things you can check beyond these three.
1. Common 17: ▲ Get Item Count
　Used in this topic. Checks how many of a specific item the player holds.
　Useful not only for checking whether the player has 1 or more of a key-type item,
　but also for events like a doctor who demands multiple healing potions.

2. Common 18: ▲ Get Weapon Count
　Checks how many of a specific weapon the player holds. Equipped weapons are not counted,
　so use "Common 21: ▲ Get Equipment" for those.
　You could create events like a swordsman asking you to give him a steel sword.

3. Common 19: ▲ Get Armor Count
　Similar to the above, but for armor. Equipped armor is not counted,
　so use "Common 21: ▲ Get Equipment" for those.
　You could create something like a soldier buying leather armor the castle lacks.

4. Common 20: ▲ Get Held Money
　Used in this topic. This time we branched on whether the player was broke or not,
　but you could also check whether they have 100 gold or more, and if so,
　deduct 100 gold and heal them or give an item — making a shop-like interaction.

5. Common 21: ▲ Get Equipment
　Checks what is equipped by a specified player character. You can specify both the equipment slot
　and whether it's a weapon or armor.
　Useful for presentations like an old man who reacts to a girl wearing her mother's keepsake.

6. Common 22: ▲ Get Member Info [Value]
　Used in this topic. Checks various stats of a specified player character, such as level, HP,
　and attack power, as well as whether they are in the party.
　You could create events like a sage who powers up the hero's special move if attack is above a threshold.

7. Common 23: ▲ Get Member Info [String]
　Checks the name or title of a specified player character.
　Among the methods introduced here, this one handles strings,
　so use "Condition (String)" when creating a branch.
　
　Variables that can only hold numeric values and string variables that hold strings are
　entirely different in nature, so the Conditional Branch is also different.
　
　Depending on how you use it, you might be able to create interesting mechanics,
　such as a bounty hunter who attacks if the character's title contains "Thief".

8. Common 24: ▲ Get Status Ailment
　Checks whether a specified player character is afflicted with a specific status ailment.
　Useful for creating a priest who can cure a curse that won't go away even after resting.

9. Common 25: ▲ Check Special Skill Possession
　Checks whether a specified player character has learned a specific special skill.
　You could create a grand mage who will only fight if at least one party member can use "Ultimate Magic".

10. Common 26: ▲ Check Combat Command Availability
　Checks whether a specified player character can use a specific combat command.
　Handy when you want a sage who powers up the hero's special technique.
　[Side Note]
■ How to branch by directly referencing the Changeable Database

　This time we used the method of "using a Common Event to look up a value, storing it in a variable,
　then reading out the value stored in the variable and branching based on the result".

　However, as you become more familiar with WOLF RPG Editor,
　you can directly specify values in Conditional Branches without going through Common Events,
　which is often more flexible and convenient.
　This method is for intermediate users and above, and involves specifying the Database.
　All of the examples given this time are information stored in the "Changeable Database",
　so the following method can do everything introduced so far.

　In the Event Command Input window, set the tab to "7 Condition (Variable)",
　then select "Changeable DB" as the variable type.
　The input fields will change to three short value entry fields,
　where you enter the Type, Data, and Field numbers from the Changeable Database.

　For example, [ 0:12:4 ] would read the value in
　Type 0 "Player Character Status", Data 12 "Wolfar", Field 4 "Level"
　(which is 5 in the default settings).

　It's a bit less intuitive, but it's a very useful method,
　so try learning it once you feel comfortable with the basics.
