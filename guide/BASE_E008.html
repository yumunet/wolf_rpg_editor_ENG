【I want to open the menu or save screen from within an event】

　[Goal]

　When a battle occurs in the middle of a long event,
　you may want to call up the menu screen to let the player prepare.
　Also, when creating a dungeon or other map where saving is only allowed at certain spots,
　you will need an event that calls up the save screen as a "save point".
　That's exactly what this topic covers.

　This time, we will create an event "that opens the menu screen when spoken to" and
　an event "that opens the save screen when spoken to",
　learning how to launch the main menu and sub-menus.

　【Steps】
■ Main Menu and Sub-Menus


　Before creating the events, let's describe the
　main menu and sub-menus of the WOLF RPG Editor Basic System.

　The main menu is the root menu that appears when you press the "X" button during the game,
　serving as the entry point for all menus.

　Sub-menus are the menus accessible from the main menu,
　such as Items, Special Skills, Equipment, Save, and System,
　which allow detailed settings.


　The main menu and sub-menus are launched differently,
　so each will be explained separately.
　■ Launching the Main Menu
[1] Create the Event

　Create an event at a suitable spot in "Sample Map A" and
　set it to the chicken image.

　If you are unsure how, refer to steps [1]–[5] of
　"I want to try creating a new event".
[2] Enter the Event Command

　Click "■ Show Command Input Window ■"
　at the bottom right of the Event window.

　① Change the tab to "I Common Event" and configure the event operation process fields.
　② Select "Insert Event" using the radio button
　　 (this is selected by default).
　③ From the dropdown list on the right, select "Common 127: X [Move] Menu Launch".
　④ Finally, press the "Input" button at the bottom right of the window.
[3] Confirm the Event

　You have now created an event that opens the menu screen when spoken to.
　Does the event content look like the image on the right?

　Close the Event Command Input window and
　do a Test Play.
　If pressing the Confirm key to talk to the event
　opens the menu screen, you have succeeded.
　Great work!
　■ Calling a Sub-Menu
　This time we will create an "event that opens the save screen when launched".

[1] Create the Event

　Create an event at a suitable spot in "Sample Map A" and
　set it to the chicken image.

　If you are unsure how, refer to steps [1]–[5] of
　"I want to try creating a new event".
[2] Enter the Event Command

　Click "■ Show Command Input Window ■"
　at the bottom right of the Event window.

　① Change the tab to "I Common Event" and configure the event operation process fields.
　② Select "Insert Event" using the radio button
　　 (this is selected by default).
　③ From the dropdown list on the right, select "Common 14: ○ Call Various Menus".
　④ Enter the process to call the save screen. In the [Common Event Input (Value)] field,
　　set "Menu to Call" to "0: Save Screen".
　⑤ Finally, press the "Input" button at the bottom right of the window.
[3] Confirm the Event

　You have now created an event that opens the save screen when spoken to.
　Does the event content look like the image on the right?

　Close the Event Command Input window and
　do a Test Play.
　If pressing the Confirm key to talk to the event
　opens the save screen, you have succeeded.
　Great work!
　[Side Note]
■ About Sub-Menus That Can Be Called

　In the steps above we called the save screen, but
　you can call other sub-menus by changing the
　value of "Menu to Call".

　The sub-menus that can be called are as follows:
　　 0 : Save screen
　　 -1 : Load screen
　　 1 : Item screen
　　 2 : Skill screen
　　 3 : Equipment screen
　　 4 : System screen
